## 二、里程碑规划

### Milestone 1：MVP 核心闭环

> **目标**：调度员能创建任务、分配司机车辆、司机能接单执行。做完这个，系统就能用。

#### Phase 1.1：项目骨架

- [x] **初始化 Monorepo** ✅
  - 开发指令：`使用 pnpm workspace 初始化项目，创建 apps/frontend、apps/server、packages/shared-types 目录结构，配置 TypeScript、ESLint、Prettier`
  - 依赖：无
  - 完成标准：`pnpm install` 成功，`pnpm dev` 能启动前端和后端

- [x] **数据库模型 + 迁移** ✅
  - 开发指令：`根据产品概述创建所有 SQLAlchemy 数据模型（user、task、vehicle、driver、warehouse），配置 Alembic 迁移`
  - 依赖：项目骨架
  - 完成标准：`alembic upgrade head` 创建所有表成功

#### Phase 1.2：基础设施

- [x] **shared 公共模块** ✅
  - 开发指令：`搭建 shared/ 目录，包含 Axios 客户端（拦截器）、logger 工具、日期/格式化工具、通用 UI 组件（EmptyState、LoadingSpinner）`
  - 依赖：项目骨架
  - 完成标准：其他模块可以直接引用 shared 的工具和组件

#### Phase 1.3：核心业务

- [x] **auth 用户认证** ✅
  - 开发指令：`开发登录页面、JWT 认证（后端生成/验证 token）、路由守卫（未登录跳转登录页）、角色权限控制（调度员/司机/管理员）`
  - 依赖：shared、数据库模型
  - 完成标准：用户能登录/登出，未登录无法访问任何页面，不同角色看到不同菜单

- [ ] **fleet 车队管理** 🔄（阶段 0~4 ✅，阶段 5 ⬜）
  - 开发指令：`开发车辆增删改查、司机增删改查、车辆状态展示（空闲/运输中/超时）、证照管理（上传/到期预警）、运输流水记录`
  - 依赖：auth、shared
  - 完成标准：能新增车辆和司机，查看列表，编辑信息，车辆状态正确显示
  - 进度：34/41 任务完成（阶段 0 基础设施 ✅ / 阶段 1 车辆管理 ✅ / 阶段 2 司机管理 ✅ / 阶段 3 证照管理 ✅ / 阶段 4 运输流水 ✅ / 阶段 5 车队统计 ⬜）

- [ ] **dispatch 调度中心**
  - 开发指令：`开发任务创建表单（客户信息、起运地/目的地、箱号箱型封号、业务类型）、任务列表（搜索/筛选/分页）、分配司机车辆（检查可用性）、任务状态流转（待分配→已分配→运输中→已完成/超时）、常用地址、批量删除/导出、OCR 箱号识别（拍照上传识别）`
  - 依赖：auth、fleet、shared
  - 完成标准：调度员能创建任务、查看列表、分配司机车辆、跟踪状态变化，OCR 拍照能识别箱号

- [ ] **driver 司机端**
  - 开发指令：`开发司机端独立 HTML 页面（登录、任务大厅、我的任务、任务详情、历史记录），司机能浏览可接任务、接单、更新任务状态为运输中/已完成`
  - 依赖：auth、dispatch
  - 完成标准：司机用手机浏览器登录后能看到任务大厅、接单、更新状态，调度端能看到状态变化

### Milestone 2：完整版

> **目标**：补齐剩余业务模块，系统功能完整。

#### Phase 2.1：业务补齐

- [ ] **warehouse 仓库管理**
  - 开发指令：`开发仓库增删改查、库位可视化网格布局（不同颜色表示空/重/空箱状态）、入库/出库操作、库存查询筛选、集装箱移动、批量导入`
  - 依赖：auth、shared
  - 完成标准：能管理仓库和库位，出入库操作正常，库位状态颜色正确

- [ ] **dashboard 运营看板**
  - 开发指令：`开发首页看板（今日任务数、完成率、超时数、平均转运时间）、天地图地图集成 + 车辆实时定位标记、地图与运力列表双向联动、30 秒自动刷新`
  - 依赖：auth、dispatch、fleet
  - 完成标准：打开首页能看到全局指标和地图上的车辆位置，点车辆能看详情

- [ ] **settings 系统设置**
  - 开发指令：`开发用户管理（增删改查）、系统参数配置、操作日志查看、调度参数设置`
  - 依赖：auth、shared
  - 完成标准：管理员能管理用户、修改系统参数、查看操作日志

## 三、风险识别

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 天地图 API 对接问题 | dashboard | 先用 Leaflet 默认图层开发，天地图 key 就绪后切换 |
| OCR 识别精度不足 | dispatch | 识别结果允许手动修正，提供重新拍照入口 |
| 司机端兼容性 | driver | 只适配主流手机浏览器（微信/Safari/Chrome），不追求全覆盖 |
| SQLite 并发瓶颈 | 全部 | M1 用 SQLite 开发，M2 后评估是否切 PostgreSQL |

## 六、开发流程与 Skills 体系

V6 项目采用 **21 个 Skills + 11 阶段** 的规范驱动开发流程。详见 [.trae/rules/guardrails.md](../.trae/rules/guardrails.md)。

### 核心工作流

```
项目级（一次性）：
  需求澄清 → 产品规划 → 技术设计 → 路线图规划 → 项目初始化

功能级（循环使用）：
  功能需求澄清 → 功能技术设计 → 任务规划 → TDD编码实现 → 测试验证
                                                            ↓
                                                      功能迭代 ←┘
```

### 常用 Skills

| 场景 | Skill | 说明 |
|------|-------|------|
| 新功能开发 | `/feature-requirements-clarification` | 澄清功能需求，产出 AC |
| 技术方案设计 | `/feature-design` | 设计 DB/API/组件结构 |
| 任务拆分 | `/task-planning` | 垂直切片，定义验收标准 |
| 编码实现 | `/feature-implementation` | TDD 驱动，穿透所有技术层 |
| 完成前验证 | `/verification-before-completion` | 声称"完成"前必须运行 |
| 分支收尾 | `/finishing-a-development-branch` | 合并/PR/保留/丢弃分支 |
