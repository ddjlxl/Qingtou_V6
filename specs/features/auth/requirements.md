# auth 用户认证 — 需求文档

> **版本**：v1.0
> **创建日期**：2026-05-04
> **状态**：草稿
> **依赖**：shared 公共模块、数据库模型（users 表）

---

## 一、功能概述

auth 模块是 V6 的用户认证系统，负责登录/登出、JWT Token 管理、路由守卫和角色权限控制。所有业务页面必须在登录后才能访问。

---

## 二、用户故事

### US-001：用户登录
> 作为调度员/司机/管理员，我希望能用用户名和密码登录系统，以便使用我的专属功能。

### US-002：Token 自动携带
> 作为已登录用户，我希望每次请求自动携带身份凭证，不需要手动填写。

### US-003：未登录拦截
> 作为系统管理员，我希望未登录用户无法访问任何业务页面，确保数据安全。

### US-004：角色权限控制
> 作为系统管理员，我希望不同角色看到不同的菜单和页面，司机不能访问调度功能。

### US-005：登出
> 作为用户，我希望能安全退出系统，清除登录状态。

---

## 三、验收标准（AC）

### AC-001：登录 API
- `POST /api/v1/auth/login` 接收 `{ username, password }`
- 验证用户名密码（bcrypt 比对）
- 检查用户状态（disabled 用户拒绝登录）
- 返回 `{ code: 200, data: { token, user: { id, username, name, role } } }`
- 更新 `last_login_at` 字段

### AC-002：获取当前用户 API
- `GET /api/v1/auth/me` 需要 Bearer Token
- 返回当前登录用户信息
- Token 无效或过期返回 401

### AC-003：登录页面
- 用户名 + 密码输入框
- 登录按钮（带 loading 状态）
- 登录失败显示错误提示
- 登录成功跳转到首页 `/`
- 调用 `frontend-design` 设计视觉品质

### AC-004：Token 存储
- 登录成功后 token 存入 `localStorage`
- 用户信息存入 auth store（Pinia）
- 页面刷新后从 localStorage 恢复 token

### AC-005：路由守卫
- 未登录用户访问任何页面 → 跳转 `/login`
- 已登录用户访问 `/login` → 跳转 `/`
- 路由守卫在 `router/index.ts` 中通过 `beforeEach` 实现

### AC-006：角色权限
- 不同角色看到不同的侧边栏菜单
- 司机角色不能访问调度端页面
- 路由层面拦截无权限访问（重定向到首页）

### AC-007：登出
- 清除 localStorage 中的 token
- 清除 auth store 中的用户信息
- 跳转到登录页

## 四、业务流程

### 4.1 登录流程

```
用户输入用户名密码 → 点击登录
  → 前端 POST /api/v1/auth/login
    → 后端查询用户 → 验证密码 → 检查状态
      → 生成 JWT Token → 更新 last_login_at
        → 返回 token + 用户信息
  → 前端存储 token → 存储用户信息 → 跳转首页
```

### 4.2 路由守卫流程

```
用户访问任意页面
  → router.beforeEach 检查
    → 有 token？
      → 是：放行（如果访问 /login 则重定向到 /）
      → 否：是否访问 /login？
        → 是：放行
        → 否：重定向到 /login
```

### 4.3 权限检查流程

```
用户访问页面
  → 路由 meta.roles 定义了允许的角色
    → 当前用户角色在允许列表中？
      → 是：放行
      → 否：重定向到首页
```

## 五、数据规则

| 规则 | 说明 |
|------|------|
| 用户名 | 必填，3-50 字符 |
| 密码 | 必填，6-255 字符，后端 bcrypt 加密存储 |
| Token 有效期 | 24 小时 |
| 禁用用户 | 可查询到但拒绝登录，提示"账号已被禁用" |
| 角色枚举 | admin / dispatcher / driver（与 shared-types UserRole 一致） |

---

## 六、非功能需求

| 需求 | 说明 |
|------|------|
| 安全性 | 密码使用 bcrypt 哈希，不存储明文 |
| 安全性 | JWT 使用 HS256 算法，密钥从环境变量读取 |
| 性能 | 登录响应时间 < 500ms |
| 体验 | 登录失败有明确错误提示（用户名不存在/密码错误/账号禁用） |
