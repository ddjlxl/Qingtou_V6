# 司机工作台移动端适配

## 功能概述
- **功能名称**：司机工作台移动端适配
- **功能描述**：将 /driver 页面从桌面端侧边栏布局改造为移动端优先的底部 Tab 布局，适配司机在手机上操作的使用场景
- **目标用户**：司机（10-30 人，手机浏览器）
- **优先级**：高

## 核心流程
司机在手机上打开系统 → 自动识别为移动端布局 → 底部显示「任务/历史/我的」三 Tab → 默认进入「任务」Tab 查看任务列表 → 通过横向滚动切换状态筛选（全部/待发车/运输中/已完成）→ 向下滚动自动加载更多任务 → 点击卡片上的操作按钮完成运输流程。

## 验收标准（AC）

### Happy Path

- **AC-001**: Given 司机在手机浏览器（屏幕宽度 < 768px）中访问系统，When 页面加载完成，Then 布局自动切换为移动端模式：顶部精简标题栏 + 底部固定三 Tab 导航栏（任务/历史/我的），左侧侧边栏不显示，顶部 header 的用户下拉和折叠按钮不显示。

- **AC-002**: Given 桌面端用户（admin 或 dispatcher 角色）在任意宽度下访问系统，When 页面加载完成，Then 始终显示桌面端侧边栏布局，不触发移动端底部 Tab 栏。

- **AC-003**: Given 司机在移动端底部栏，When 点击「任务」Tab，Then 进入 /driver 页面，显示任务列表。

- **AC-004**: Given 司机在移动端底部栏，When 点击「历史」Tab，Then 进入 /driver/history 页面，显示占位空状态（"暂无历史任务"）。

- **AC-005**: Given 司机在移动端底部栏，When 点击「我的」Tab，Then 进入 /driver/profile 页面，显示司机姓名（从当前登录信息获取）和「退出登录」按钮。

- **AC-006**: Given 司机在移动端「任务」页面，When 查看任务卡片，Then 每张卡片占满屏幕宽度（单列布局），信息行纵向排列，不再使用桌面端的两列网格布局。

- **AC-007**: Given 司机在移动端查看任务卡片，When 卡片包含操作按钮（"开始运输"或"完成任务"），Then 按钮高度至少 44px，宽度至少占卡片宽度的 40%，方便手指点击。

- **AC-008**: Given 司机在移动端「任务」页面有超过 20 条任务，When 向下滚动到列表底部，Then 自动加载下一页任务数据，新数据追加到列表尾部，不替换已有数据。

- **AC-009**: Given 司机在移动端切换状态筛选 Tab（全部/待发车/运输中/已完成），When 点击某个 Tab，Then 重置列表并重新从第一页加载，Tab 在窄屏下支持横向滚动。

- **AC-010**: Given 司机点击「我的」页面的「退出登录」按钮，When 确认退出，Then 清除登录状态并跳转到 /login 页面。

### Edge & Error Cases

- **AC-011**: Given 司机在移动端上拉加载更多时网络异常，When 加载请求失败，Then 显示"加载失败"提示，已加载的数据保留在列表中不丢失。

- **AC-012**: Given 司机在移动端上拉加载更多时已是最后一页，When 滚动到底部，Then 显示"没有更多了"提示，不再触发加载请求。

- **AC-013**: Given 司机在移动端正处于加载中状态，When 再次快速滚动到底部，Then 不重复触发加载请求（防抖）。

- **AC-014**: Given 司机在移动端某个状态 Tab 下没有任务，When 切换到该 Tab，Then 显示 EmptyState 空状态组件（"暂无任务"）。

- **AC-015**: Given 司机在移动端切换状态 Tab 时正在加载中，When 快速点击多个 Tab，Then 只保留最后一次点击对应的请求结果，避免数据错乱。

### Business Rules

- **AC-016**: Given 任意角色用户，When 屏幕宽度 < 768px 且用户角色为 driver，Then 触发移动端布局，两个条件缺一不可。

- **AC-017**: Given 移动端布局，When 底部栏渲染，Then 始终固定在屏幕底部（position: fixed），不随页面滚动而移动，三个 Tab 均分宽度。

- **AC-018**: Given 移动端任务卡片操作按钮，When 任务状态为"待发车"，Then 显示"开始运输"按钮；When 任务状态为"运输中"，Then 显示"完成任务"按钮；When 任务状态为"已完成"，Then 不显示任何操作按钮。

- **AC-019**: Given 移动端任务卡片，When 显示任务信息，Then 包含：任务编号、状态标签、客户名称、路线（起运地 → 目的地）、箱号、空重箱、车牌号。

## 范围界定

### 本次做
- AppLayout.vue 响应式改造：宽度 + 角色双重判断，移动端底部三 Tab 栏
- DriverWorkbench.vue 移动端适配：卡片单列全宽、按钮 44px+ 触控区、状态 Tab 横向滚动
- useDriverStore.ts 改造：新增 hasMore 计算属性 + loadMore 追加加载方法
- 新增两个占位路由：/driver/history、/driver/profile（极简占位组件）
- 翻页改为上拉无限滚动（v-infinite-scroll）

### 本次不做
- 「历史任务」的完整功能实现（仅占位）
- 「我的」页面的完整个人信息编辑功能（仅显示姓名 + 退出登录）
- 其他页面（fleet、dispatch）的移动端适配
- 移动端的 PWA、离线缓存、原生 App 封装
- 暗黑模式或其他主题适配
