# 仓库管理

## 功能概述
- **功能名称**：仓库管理
- **功能描述**：库管对仓库 12 个区域（共 144 个库位）进行集装箱入库、出库、移动和查询管理，出库自动联动调度中心创建任务
- **目标用户**：库管（独立角色，电脑浏览器）
- **优先级**：高

## 核心流程
库管进入仓库页面，看到 12 个区域的 3×4 库位网格和全局统计。集装箱可通过两种方式入库：调度任务完成后自动入库（按优先级自动分配库位），或库管手动录入/导入（自选区域）。库管可在库位间移动集装箱，点出库后系统自动在调度中心创建"待分配"任务。页面支持按状态筛选和关键词搜索，操作后统计数据实时更新。

## 验收标准（AC）

### Happy Path
- **AC-001**：Given 调度员或司机标记任务为"已完成"，且仓库有可用空位，When 任务完成时，Then 系统将任务中每个集装箱各分配一个库位，按优先级依次分配。任务有多个箱号时，每个箱号各占一个库位。
- **AC-002**：Given 库管在仓库管理页面，When 选择目标区域并手动录入或导入集装箱数据（箱号必填，重箱/空箱状态必选），且目标区域有可用空位，Then 集装箱成功入库，占据该区域按行优先从左到右（(1,1)→(1,4)→(2,1)→...→(3,4)）的第一个可用空位。
- **AC-003**：Given 库管选中一个或多个有箱库位，When 点击"出库"并确认，Then 库位变空，系统自动在调度中心创建"待分配"任务，已填入库时录入的集装箱信息（箱号必填，客户/箱型/封号如有则填入）。业务类型可选填，调度员后续可改。
- **AC-004**：Given 库管在仓库管理页面，When 点击"移动"按钮进入移动模式（按钮高亮），选择源库位（有箱，显示蓝色闪烁边框）再选择目标空位（显示绿色虚线边框），Then 集装箱移动到目标库位，源库位变为空位，保持移动模式可继续移动。点击"取消移动"按钮或按 ESC 键退出移动模式。
- **AC-005**：Given 库管在仓库管理页面，When 按状态筛选（全部/重箱/空箱/空位），Then 列表即时过滤，只显示匹配的库位。
- **AC-006**：Given 库管在搜索框输入箱号或货主关键词，When 输入停止 300ms 后，Then 匹配的库位高亮显示，区域标题显示匹配数量。
- **AC-007**：Given 库管进入仓库管理页面，When 页面加载完成，Then 右侧面板显示全局统计（库位总数 144、已用库位、剩余库位、重箱数、空箱数、利用率进度条）。
- **AC-008**：Given 仓库管理页面已加载，When 查看页面布局，Then 按 4 行展示 12 个区域卡片（第 1 行 3-5/3-7/3-20，第 2 行 2-13/1-1/1-20，第 3 行 6-1/6-2/6-3，第 4 行 6-4/6-5/6-6），每个卡片内为 3×4 库位网格，区域标题显示利用率（如 "3-20 区 (5/12)"）。

### Edge & Error Cases
- **AC-009**：Given 全部 12 个区域均无可用空位，When 自动入库触发时，Then 入库失败，提示"库位已满，无法入库"。
- **AC-010**：Given 库管选择的目标区域已满，When 手动入库提交时，Then 拒绝，提示"XX 区域已满，请选择其他区域"。
- **AC-011**：Given 库管手动输入箱号，When 箱号格式不符合"4位大写字母+7位数字"（如 ABCD1234567），Then 拒绝提交，提示"箱号格式错误"。
- **AC-012**：Given 库管导入或手动输入已存在的箱号，When 提交时，Then 拒绝，提示"箱号已存在"。
- **AC-013**：Given 库管导入或手动输入的数据条数超过目标区域可用空位数，When 提交时，Then 拒绝，提示"可用库位不足，当前可用 X 个，需要 Y 个"。
- **AC-014**：Given 集装箱已入库，When 调度员删除对应的已完成任务，Then 仓库中的集装箱不受影响。
- **AC-015**：Given 库管点击单个空位，When 该库位为空，Then 触发手动录入对话框。
- **AC-016**：Given 库管选中有箱库位，When 点击编辑，Then 可修改货主和备注信息并保存。

### Business Rules
- **AC-017**：Given 自动入库，When 系统分配库位时，Then 先按优先级 3-20 → 1-20 → 6-1 → 6-2 → 6-3 → 6-4 → 6-5 → 6-6 分配；优先区域全满后，依次分配到 3-5 → 3-7 → 2-13 → 1-1；全部 12 区域均满则拒绝。每个区域内按行优先从左到右分配第一个空位。
- **AC-018**：Given 移动操作，When 集装箱从源库位移到目标库位，Then 集装箱状态不变，货主和箱信息不变。
- **AC-019**：Given 任何库位状态变更，When 操作完成后，Then 统计面板和区域卡片利用率实时更新。
- **AC-020**：Given 非 warehouse_keeper 角色登录，When 访问仓库管理页面，Then 无权限访问。warehouse_keeper 为 auth 角色枚举中新增的角色。
- **AC-021**：Given 出库操作，When 出库弹窗打开，Then 业务类型下拉框可选填，不选则调度任务中该字段为空。

## 范围界定

### 本次做
- 12 区域固定布局，每个区域 3×4 共 144 个库位
- 自动入库（调度任务完成后，按优先级分配库位）
- 手动入库（库管选区域、手动录入或文件导入）
- 出库（自动创建调度中心"待分配"任务，填入库时录入的集装箱信息）
- 库内集装箱移动
- 按状态筛选（全部/重箱/空箱/空位）
- 关键词搜索（箱号、货主），300ms 防抖
- 全局统计面板 + 区域卡片利用率
- 编辑货主和备注
- 库管角色权限控制（新增 warehouse_keeper 角色）

### 本次不做

| 不做项 | 原因 | 何时考虑 |
|--------|------|---------|
| 区域/库位数量的动态配置 | 当前仓库物理布局固定（12 区域 × 12 库位），管理员无动态调整需求。动态配置需额外 Zone/Slot CRUD + 迁移脚本，复杂度高但当前无业务价值 | 第二期：当仓库扩建或需要新增区域时 |
| 删除调度任务后反向同步删除仓库数据 | 集装箱入库后可能已被库管移动或二次出库，反向同步逻辑复杂且容易导致数据不一致。入库是单向写入，不做反向关联 | 第三期：如需可选开关控制，需先定义冲突处理策略 |
| 库位预约功能 | 当前业务流程中无预约场景，集装箱入库都是即时操作。预约需额外状态（reserved）和超时释放逻辑 | 第二期：当业务出现"提前锁定库位"需求时 |
| 库存预警 | 需要定义预警规则（在库天数阈值、利用率阈值）、通知机制（站内信/推送），当前无明确业务规则 | 第二期：当库管提出具体预警需求后 |
| 修改已入库集装箱的箱号或箱型 | 箱号是集装箱唯一标识，入库后修改会导致出库创建任务时数据不一致。箱型同理，应入库时一次录对 | 第二期：如确有录入错误场景，可考虑"删除重录"而非"修改" |
| 多仓库支持 | 当前只有一个仓库，多仓库需新增 Warehouse 实体 + 切换逻辑 + 权限隔离，无业务需求 | 第二期：当公司有多个仓库需要管理时 |
| 出库时自动推断业务类型 | 空箱不一定都是"还箱"，重箱不一定都是"送货"，自动推断容易出错。让库管或调度员手动选择更可靠 | 第三期：当积累了足够数据可做智能推荐时 |

## 仓库布局规格

### 区域排列（4 行）
| 行 | 区域 |
|----|------|
| 第 1 行 | 3-5, 3-7, 3-20 |
| 第 2 行 | 2-13, 1-1, 1-20 |
| 第 3 行 | 6-1, 6-2, 6-3 |
| 第 4 行 | 6-4, 6-5, 6-6 |

### 库位网格
- 每个区域内部：3 行 × 4 列 = 12 个库位
- 总库位数：12 × 12 = 144
- 库位编号规则：按行优先从左到右，(1,1)→(1,2)→(1,3)→(1,4)→(2,1)→...→(3,4)

### 自动入库优先级
1. 优先区域：3-20 → 1-20 → 6-1 → 6-2 → 6-3 → 6-4 → 6-5 → 6-6
2. 剩余区域：3-5 → 3-7 → 2-13 → 1-1
3. 全部满则拒绝入库

## 数据验证规则

### 箱号格式
- 规则：4位大写字母 + 7位数字
- 示例：ABCD1234567
- 不允许重复

### 集装箱状态
- 必填，可选值：heavy（重箱）、empty（空箱）
- 不允许 heavy ↔ empty 直接转换

### 容量验证
- 导入/手动录入数量 ≤ 目标区域可用空位数
- 超出时显示具体差值

## 文件导入规格

### 支持格式
- Excel（.xlsx）

### 列定义
| 列序号 | 列名 | 必填 | 说明 |
|--------|------|------|------|
| 1 | 箱号 | 是 | 4位大写字母+7位数字 |
| 2 | 状态 | 是 | heavy 或 empty |
| 3 | 客户 | 否 | 货主名称 |
| 4 | 箱型 | 否 | 如 20GP、40HQ |
| 5 | 封号 | 否 | 封条号 |

## 设计决策

> 以下为需求澄清阶段讨论确认的关键决策，记录决策结论和理由，避免后续重复讨论。

### 决策 1：仓库结构采用预设方案（方案 B）

| 选项 | 说明 | 结论 |
|------|------|------|
| 方案 A：可配置 | 管理员自建区域、配置行列数 | 不选 |
| 方案 B：预设 | 区域和库位通过迁移脚本一次性写入 | **选择** |

**理由**：方案 B 只需 Slot 一张表，无需 Zone CRUD 和前端区域管理界面，实现简单。当前仓库物理布局固定，无动态调整需求。

### 决策 2：调度/司机完成任务后自动入库

**结论**：调度员和司机标记任务为"已完成"时，系统自动将集装箱写入仓库库位。

**理由**：
- 司机和调度员走的是同一个 `complete_order` 接口，无需额外开发司机端入库逻辑
- 自动入库与"完成→自动记运输流水"是同一模式，复用已有架构

### 决策 3：入库是单向写入，不做反向同步

**结论**：集装箱入库后与调度任务解耦。调度员删除已完成任务，仓库数据不受影响。

**理由**：
- 集装箱入库后可能已被库管移动或二次出库，反向同步逻辑复杂且容易导致数据不一致
- 单向写入简单可靠，仓库数据独立于调度任务生命周期

### 决策 4：库位满了直接拒绝入库，不做溢出

**结论**：全部 12 个区域均无空位时，入库失败并提示"库位已满，无法入库"。不采用"数据入库但无库位"的溢出方案。

**理由**：溢出方案需要额外的"未分配库位"状态和管理界面，增加复杂度。实际业务中 144 个库位足够，满了就提示更直观。

### 决策 5：出库自动创建调度任务，只填已有信息

**结论**：出库时自动在调度中心创建"待分配"任务，填入库时已录入的集装箱信息（箱号、客户、箱型、封号），其他字段（路线、业务类型等）留给调度员补充。

**理由**：
- 箱号、客户、箱型、封号是集装箱的"身份证"，入库时就该有，出库时直接带入减少调度员工作量
- 路线和业务类型是调度决策，不应由库管决定
- 业务类型可选填，不设默认值（空箱不一定"还箱"，重箱不一定"送货"）

### 决策 6：库管为独立角色

**结论**：在 auth 角色枚举中新增 `warehouse_keeper` 角色，非该角色无权访问仓库管理页面。

**理由**：库管和调度员职责不同，调度员管任务分配，库管管库位操作，权限应隔离。

### 决策 7：自动入库优先级分配链

**结论**：优先区域（3-20 → 1-20 → 6-1 → 6-2 → 6-3 → 6-4 → 6-5 → 6-6）→ 剩余区域（3-5 → 3-7 → 2-13 → 1-1）→ 全满拒绝。

**理由**：优先区域是业务指定的常用存放区，优先分配减少库管整理工作量。优先区域满后自动降级到剩余区域，不直接拒绝。

## 后续迭代规划

> 以下功能不在本次开发范围，按优先级排列，供后续迭代参考。

### 第二期：仓库管理增强
- 区域/库位数量的动态配置（管理员可新增/删除区域，调整行列数）
- 多仓库支持（切换不同仓库查看）
- 修改已入库集装箱的箱号或箱型
- 库位预约功能（预留库位给即将到达的集装箱）
- 库存预警（在库天数超限提醒、利用率超阈值告警）

### 第三期：智能化与集成
- 出库时根据集装箱状态自动推断业务类型
- 删除调度任务后反向同步仓库数据（可选开关）
- 仓库操作日志（谁在什么时间做了什么操作）
- 与调度中心深度联动（任务详情中显示集装箱当前库位）
- 仓库布局拖拽调整
