## 功能概述
- **功能名称**：Dispatch-Fleet 联动 & 司机端
- **功能描述**：打通调度任务与运输流水的数据壁垒——任务完成后自动记流水、任务删除同步清除流水；为司机提供极简手机端工作台
- **目标用户**：调度员、管理员、司机
- **优先级**：高

## 核心流程

**任务→流水联动**：调度端或司机端标记任务完成 → 系统自动在运输流水中创建一条记录（含订单号、客户名称、空重箱状态、启运地、目的地、箱号、执行车辆、执行司机）。若任务无司机/车辆则不创建流水。

**任务删除→流水同步删除**：调度端删除任务 → 系统按 order_no 查找关联的运输流水记录并同步删除；无关联记录则静默跳过。

**外部导入**：调度员仍可通过文件导入运输流水，导入格式从 7 列升级为 8 列（新增空重箱状态），向下兼容 7 列旧格式。

**司机登录**：手机号+密码登录，与调度员走同一套 JWT 认证体系。创建司机时自动生成登录账号（username=手机号, 初始密码=手机号）。

**司机工作台**：登录后看到分配给自己的任务列表，可执行「开始运输」「标记完成」两个操作。

## 验收标准（AC）

### Happy Path

#### 联动——完成→流水

- **AC-001**: Given 调度员在调度中心对已分配/运输中/已超时任务点击"标记完成"，Then 任务状态变为已完成，同时自动在运输流水中创建一条记录（含订单号、客户名称、空重箱状态、启运地、目的地、箱号、执行车辆、执行司机）。

- **AC-002**: Given 司机在司机端对运输中任务点击"标记完成"，Then 任务状态变为已完成，同时自动在运输流水中创建一条记录。

#### 联动——删除→同步删除

- **AC-003**: Given 调度员删除某任务，Then 若该任务已关联运输流水记录（按 order_no 匹配），同步删除该流水记录；若无关联系记录则静默跳过，任务正常删除。

#### 司机端

- **AC-004**: Given 司机输入手机号+密码，When 点击登录，Then 登录成功，进入独立的司机工作台页面（无车队管理/调度中心侧边栏菜单）。

- **AC-005**: Given 司机进入工作台，When 页面加载，Then 分页显示分配给自己的任务列表（含任务编号、客户、起运地→目的地、箱号、状态），按创建时间倒序排列。

- **AC-006**: Given 司机对"已分配"状态的任务，When 点击"开始运输"，Then 任务状态变为"运输中"。

- **AC-007**: Given 司机对"运输中"状态的任务，When 点击"标记完成"，Then 任务状态变为"已完成"，同时自动在运输流水中创建一条记录（含订单号、客户名称、空重箱状态、启运地、目的地、箱号、执行车辆、执行司机）。

#### 外部导入

- **AC-008**: Given 调度员在运输流水页导入文件，When 文件为 8 列格式（含空重箱状态），Then 成功导入全部字段。When 文件为 7 列旧格式，Then 兼容导入（空重箱状态留空）。

- **AC-009**: Given 导入文件中车牌号在系统中不存在，Then 该行报错"车牌号 XX 不存在"，其他行正常导入。

- **AC-010**: Given 导入文件中司机手机号在系统中不存在，Then 该行报错"手机号 XX 对应的司机不存在"，其他行正常导入。

### Edge & Error Cases

- **AC-011**: Given 完成任务时 driver_id 或 vehicle_id 为空（未分配司机/车辆的任务被标记完成），Then 跳过运输流水创建，不报错，任务正常完成。

- **AC-012**: Given 删除任务时该任务无关联运输流水记录，Then 静默跳过，任务正常删除。

- **AC-013**: Given 已完成的任务被再次调用完成接口，Then 后端拒绝（当前状态不允许标记完成），不会重复创建运输流水。

- **AC-014**: Given 导入文件中某 order_no 已存在于运输流水表，Then 跳过该行，统计为重复记录，其他行正常导入。

- **AC-015**: Given 导入文件中空重箱状态值无效（非 heavy/empty/空字符串），Then 该行报错"空重箱状态值无效，应为 heavy 或 empty"。

- **AC-016**: Given 调度员修改司机手机号，Then 同步更新对应 User 的 username 和 phone（按旧 phone 匹配）。

- **AC-017**: Given 调度员删除司机（无历史记录），Then 同步删除对应的 User 登录账号。

- **AC-018**: Given 调度员停用司机，Then 同步禁用对应的 User 账号。

- **AC-019**: Given 司机尝试操作不属于自己的任务，Then 返回 403 错误"这不是您的任务"。

- **AC-020**: Given 司机对非"已分配"状态的任务点击"开始运输"，Then 返回 422 错误"仅已分配状态的任务可开始运输"。

### Business Rules

- **AC-021**: 运输流水的 order_no 与调度任务的 order_no 一一对应，通过 order_no 唯一约束保证不重复。

- **AC-022**: 空重箱状态由调度员在创建/编辑任务时通过 container_status 字段显式指定（heavy/empty），与 business_type 独立。短驳任务也需指定空重箱状态。

- **AC-023**: 司机只能操作分配给自己的任务（driver_id 匹配当前司机），不能查看或操作他人任务。

- **AC-024**: 司机创建时自动生成 User 账号（role=driver, username=phone, password=hashed(phone)），修改/删除/停用司机时同步维护 User 账号。

## 数据规格

### 新增字段：Order.container_status
- **字段名**：container_status
- **类型**：字符串，可选（nullable）
- **枚举**：heavy（重箱）/ empty（空箱）
- **说明**：独立于 business_type，专门用于运输流水统计

### 新增字段：TransportRecord.container_status
- **字段名**：container_status
- **类型**：字符串，可选（nullable）
- **枚举**：heavy（重箱）/ empty（空箱）
- **说明**：从 Order.container_status 映射或从导入文件读取

### 司机登录账号
- **username**：手机号
- **password**：初始密码=手机号（bcrypt 加密存储）
- **role**：driver

### 运输流水导入格式（v2）
8 列，Tab 分隔：
```
任务编号	客户信息	空重箱状态	起运地	目的地	箱号	执行车辆(车牌号)	执行司机(手机号)
```
向下兼容 7 列旧格式（空重箱状态视为空）。
