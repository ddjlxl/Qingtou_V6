# 任务箱号封号补充编辑

## 功能概述
- **功能名称**：任务箱号封号补充编辑
- **功能描述**：允许调度员对已分配/运输中/已超时状态的任务编辑箱号和封号，解决提空箱任务创建时箱号留空、司机到场后需补填的问题
- **目标用户**：调度员
- **优先级**：高

## 核心流程
提空箱任务创建时箱号留空 → 分配司机 → 司机到场提箱 → 电话/微信告知调度员箱号 → 调度员在调度中心编辑任务补填箱号和封号。复用现有编辑弹窗，非待分配状态时仅箱号和封号字段可编辑，其他字段禁用。

## 验收标准（AC）

### Happy Path
- **AC-001**: Given 某任务状态为"已分配"、"运输中"或"已超时"，When 调度员点击"编辑"按钮，Then 弹出编辑弹窗，箱号和封号字段可编辑，其他字段禁用（灰色不可点击）。
- **AC-002**: Given 调度员在编辑弹窗中填写了箱号，When 点击保存，Then 箱号自动转大写，保存成功，任务列表中箱号列更新显示。
- **AC-003**: Given 调度员在编辑弹窗中填写了封号，When 点击保存，Then 封号自动转大写，保存成功。

### Edge & Error Cases
- **AC-004**: Given 某任务状态为"已完成"，When 调度员查看操作按钮，Then 不显示"编辑"按钮。
- **AC-005**: Given 调度员编辑已分配/运输中/已超时任务时填入的箱号与另一个进行中任务的箱号重复，When 点击保存，Then 提示"该箱号已有进行中的任务，请检查"，保存失败。
- **AC-006**: Given 调度员编辑已分配/运输中/已超时任务时填入的箱号格式不正确（非4位大写字母+7位数字），When 点击保存，Then 提示格式错误，保存失败。

### Business Rules
- **AC-007**: 已分配/运输中/已超时状态仅允许编辑箱号和封号两个字段，其他字段（客户、路线、箱型、业务类型、司机、车辆、备注等）不可编辑。
- **AC-008**: 待分配状态的任务编辑行为不变，仍可编辑所有字段。

## 范围界定

### 本次做
- 后端放宽编辑限制：已分配/运输中/已超时状态允许编辑箱号和封号
- 前端复用现有编辑弹窗，已分配/运输中/已超时状态时只启用箱号和封号字段，其他字段禁用

### 本次不做
- 司机端箱号回填（后续 OCR 功能一起做）
- OCR 拍照识别箱号
- 改派（换司机/车辆）
- 已完成状态的任务编辑
