## 功能概述
- **功能名称**：Dispatch 调度中心
- **功能描述**：调度员的核心工作台，管理运输任务的全生命周期——创建任务、分配司机车辆、跟踪状态变化
- **目标用户**：调度员、管理员
- **优先级**：高（MVP 核心模块）

## 核心流程

**任务创建流程**：调度员可先创建一个骨架任务（字段大多可选，后续补全）。选择业务类型可快速填充常用的起运地/目的地。填写路线（起运地 + 途径点 + 目的地）、集装箱信息（箱号/箱型/封号）、客户信息、单证勾选后，选择「创建任务」进入待分配列表，或选择「创建并派车」直接分配司机车辆。提空箱时箱号可留空，司机到场后回填。

**任务分配流程**：调度员在待分配任务列表中选中任务，从空闲司机车辆列表中选择，确认后任务状态变为「已分配」，车辆状态自动变为「运输中」。分配操作涉及多步（检查可用性 → 创建任务关联 → 更新车辆状态），需保证原子性：任一步失败则整体回滚。

**状态流转**：待分配 → 已分配 → 运输中 → 已完成。任务分配后超过 4 小时未完成，自动标记为「已超时」（定时任务每分钟检测一次）。

**任务编辑**：仅「待分配」状态的任务可编辑所有字段。其他状态不可编辑，如需更换司机车辆，删除任务后重新创建。

**任务删除**：所有状态均可删除。待分配直接删除；已分配/运输中/已完成/已超时需二次确认，删除时自动释放关联的车辆和司机。

## 验收标准（AC）

### Happy Path（正常流程）

#### 任务创建

- **AC-001**: Given 调度员在调度中心页面，When 点击「新建任务」按钮，Then 弹出任务创建弹窗，包含客户信息、路线规划、集装箱信息、业务类型、单证勾选区域。

- **AC-002**: Given 调度员在创建弹窗中，When 未填写任何字段直接点击「创建任务」，Then 任务创建成功，弹窗关闭，任务出现在列表中，状态显示为「待分配」，未填字段在列表中显示为空或「-」。调度员可后续通过编辑补全信息。

- **AC-003**: Given 调度员在创建弹窗中填写完整信息并选择了司机和车辆后点击「创建并派车」，Then 任务创建成功且同时完成分配，任务状态显示为「已分配」，车辆状态变为「运输中」。

- **AC-004**: Given 调度员在创建弹窗中，When 未选择司机或车辆，Then 「创建并派车」按钮置灰不可点击，hover 时提示「请选择司机和车辆」。

- **AC-005**: Given 调度员在路线规划区域，When 点击「添加途径点」并输入地址，Then 途径点被添加到路线中，可添加多个途径点，路线显示为「起运地 → 途径点1 → 途径点2 → 目的地」。

- **AC-006**: Given 调度员在集装箱信息区域，When 选择箱型下拉框，Then 显示选项：20GP、40GP、40HQ、45HQ。

- **AC-007**: Given 调度员在业务类型区域，When 选择业务类型（重箱运输 / 空箱运输 / 短驳），Then 系统自动填入该业务类型上一次使用的起运地和目的地（如无历史记录则留空），调度员可手动修改。每次创建任务时，系统记录本次使用的起运地/目的地作为该业务类型的「上一次」。

- **AC-008**: Given 调度员在单证勾选区域，When 勾选「提箱单」「过磅」「整改」中的一项或多项，Then 勾选状态被记录，任务创建后单证信息在任务列表中可见。

#### 任务列表

- **AC-009**: Given 调度员进入调度中心页面，When 页面加载完成，Then 以 el-table 表格形式展示任务列表，包含列：任务编号、客户名称、路线（起运地 → 途径点 → 目的地）、箱号、箱型、业务类型、单证（图标标签）、司机/车牌（上下两行合并显示）、状态、创建时间、操作。无途径点时路线显示为「起运地 → 目的地」。

- **AC-010**: Given 调度员在搜索框中输入关键词，When 输入完成后 300ms，Then 列表自动筛选，同时匹配任务编号、箱号、客户名称、车牌号中包含该关键词的任务。

- **AC-011**: Given 调度员在状态 Tab 栏，When 点击某个状态 Tab（全部/待分配/已分配/运输中/已完成/已超时），Then 列表只显示对应状态的任务，每个 Tab 上显示该状态的任务数量。

- **AC-012**: Given 任务数量超过一页，When 调度员查看列表底部，Then 显示分页控件，支持切换每页条数和跳转页码。

- **AC-013**: Given 任务列表为空，When 调度员查看页面，Then 显示空状态提示和「新建任务」按钮。

#### 任务分配

- **AC-014**: Given 调度员在待分配任务列表中，When 点击某任务的「分配」按钮，Then 弹出分配弹窗，显示空闲司机和车辆列表供选择。

- **AC-015**: Given 调度员在分配弹窗中选择司机和车辆后点击「确认分配」，Then 任务状态变为「已分配」，关联的车辆状态变为「运输中」。

- **AC-016**: Given 调度员在分配弹窗或创建弹窗中，When 选择一辆车，Then 系统自动填入该车辆绑定的司机（如有）；When 选择一位司机，Then 系统自动填入该司机绑定的车辆（如有）。自动填入后可手动修改。

#### 状态流转

- **AC-017**: Given 某任务状态为「已分配」，When 司机在司机端将任务状态更新为「运输中」，Then 调度端该任务状态同步变为「运输中」。

- **AC-018**: Given 某任务状态为「运输中」，When 司机在司机端将任务标记为「已完成」，Then 调度端该任务状态同步变为「已完成」，关联车辆状态恢复为「空闲」。

- **AC-019**: Given 某任务状态为「已分配」「运输中」或「已超时」，When 调度员在任务列表点击「标记完成」，Then 任务状态变为「已完成」，关联车辆状态恢复为「空闲」。

#### 任务编辑

- **AC-020**: Given 某任务状态为「待分配」，When 调度员点击「编辑」按钮，Then 弹出编辑弹窗，回填所有已填字段，可修改任意字段后保存。编辑时触发与创建相同的校验规则（起运地≠目的地、箱号唯一性、箱号格式）。

- **AC-021**: Given 某任务状态为「已分配」「运输中」「已完成」或「已超时」，When 调度员查看操作按钮，Then 不显示「编辑」按钮。

#### 任务删除

- **AC-022**: Given 某任务状态为「待分配」，When 调度员点击「删除」按钮，Then 任务直接从列表中移除。

- **AC-023**: Given 某任务状态为「已分配」或「运输中」，When 调度员点击「删除」按钮，Then 弹出确认弹窗「该任务正在执行中，确定删除吗？车辆和司机将自动释放」，确认后任务删除，关联车辆和司机恢复为空闲状态。

- **AC-024**: Given 某任务状态为「已完成」或「已超时」，When 调度员点击「删除」按钮，Then 弹出确认弹窗「确定删除该任务吗？」，确认后任务从列表中移除。

#### 常用地址

- **AC-025**: Given 调度员在路线规划区域，When 点击「常用地址」入口，Then 弹出常用地址管理弹窗，显示已保存的地址列表。

- **AC-026**: Given 调度员在常用地址弹窗中，When 输入地址名称后点击「保存」，Then 地址被添加到常用地址列表中。

- **AC-027**: Given 调度员在常用地址列表中，When 点击某地址的「设为起运地」或「设为目的地」，Then 该地址自动填入任务创建表单的对应字段。

- **AC-028**: Given 调度员在常用地址列表中，When 点击某地址的「删除」，Then 该地址从列表中移除。

### Edge & Error Cases（边界和异常）

#### 任务创建

- **AC-029**: Given 调度员创建任务时同时填写了起运地和目的地，When 起运地和目的地相同，Then 系统拒绝创建，提示「起运地和目的地不能相同」。若任一方未填写则不触发此校验。

- **AC-030**: Given 调度员创建任务时填写了箱号，When 该箱号已存在于其他进行中的任务（待分配/已分配/运输中），Then 系统拒绝创建，提示「该箱号已有进行中的任务，请检查」。箱号为空时不触发此校验。

- **AC-031**: Given 调度员创建任务时填写了箱号，When 箱号格式不正确（非 4 位大写字母 + 7 位数字），Then 箱号字段下方显示格式错误提示。箱号为空时不触发此校验。

#### 任务分配

- **AC-032**: Given 调度员分配任务时，When 选择的司机已有进行中的任务，Then 该司机在列表中显示为不可选（灰色），无法被分配。

- **AC-033**: Given 调度员分配任务时，When 选择的车辆状态为「运输中」或「超时」，Then 该车辆在列表中显示为不可选（灰色），无法被分配。

- **AC-034**: Given 调度员打开分配弹窗，When 当前没有空闲司机或车辆，Then 列表显示空状态，提示「暂无可分配的司机/车辆」。

#### 任务删除

- **AC-035**: Given 调度员删除任务时弹出确认弹窗，When 点击「取消」，Then 弹窗关闭，任务不被删除，状态不变。

#### 常用地址

- **AC-036**: Given 调度员在常用地址弹窗中，When 输入的地址名称已存在，Then 系统拒绝保存，提示「该地址已存在」。

### Business Rules（业务规则）

- **AC-037**: Given 任何情况下，When 任务状态由系统自动流转，Then 必须按路径：待分配 → 已分配 → 运输中 → 已完成，不允许跳过中间状态。调度员手动「标记完成」不受此限制，可从已分配/运输中/已超时直接标记为已完成。

- **AC-038**: Given 某任务状态为「已分配」，When 从分配时间起超过 4 小时仍未完成，Then 系统自动将该任务标记为「已超时」，关联车辆状态变为「超时」。

- **AC-039**: Given 调度员删除任何关联了车辆和司机的任务，When 删除操作完成，Then 关联的车辆状态恢复为「空闲」，关联的司机释放（可被新任务分配）。

- **AC-040**: Given 调度员输入箱号，When 输入完成，Then 箱号自动转换为大写字母。

- **AC-041**: Given 调度员输入封号，When 输入完成，Then 封号自动转换为大写字母。

- **AC-042**: Given 调度员或管理员登录系统，When 访问调度中心模块，Then 两者权限相同，均可对任务进行增删改查和分配操作。

## 数据规格

### 任务信息
- **任务编号**：自动生成，唯一，字符串
- **客户名称**：可选，字符串（拉别人的箱时不需要）
- **联系电话**：可选，字符串
- **起运地**：可选，字符串（不确定时可留空，后续补全）
- **目的地**：可选，字符串（不确定时可留空，后续补全）
- **途径点**：可选，字符串数组，可包含多个
- **箱号**：可选，字符串，格式为 4 位大写字母 + 7 位数字，自动转大写（提空箱时可留空，司机到场后回填）
- **封号**：可选，字符串，自动转大写
- **箱型**：可选，枚举（20GP / 40GP / 40HQ / 45HQ）
- **业务类型**：可选，枚举（重箱运输 / 空箱运输 / 短驳），选择后自动填入该类型上一次使用的起运地/目的地（后端存储，无历史则留空）
- **单证**：可选，多选（提箱单 / 过磅 / 整改）
- **关联车辆**：可选，关联车辆表 ID
- **关联司机**：可选，关联司机表 ID
- **调度员**：自动填充当前登录用户
- **调度备注**：可选，多行文本
- **状态**：枚举（待分配 / 已分配 / 运输中 / 已完成 / 已超时）
- **分配时间**：自动生成，分配司机车辆时记录
- **超时阈值**：分配后超过 4 小时未完成，自动标记为「已超时」
- **创建时间**：自动生成

### 常用地址
- **地址名称**：必填，唯一，字符串

## 页面结构

### 单页面布局

```
调度中心（路由：/dispatch）
├── 顶部操作栏
│   ├── 搜索框（任务编号/箱号/客户/车牌，300ms 防抖）
│   └── 新建任务按钮
├── 状态 Tab 栏
│   ├── 全部（N）
│   ├── 待分配（N）
│   ├── 已分配（N）
│   ├── 运输中（N）
│   ├── 已完成（N）
│   └── 已超时（N）
├── 任务列表（el-table）
│   ├── 列：任务编号、客户名称、路线（起运地 → 途径点 → 目的地）、箱号、箱型、业务类型、单证（图标）、司机/车牌（合并列）、状态、创建时间、操作
│   └── 操作按钮：分配（待分配）/ 编辑（待分配）/ 标记完成（已分配、运输中、已超时）/ 删除
└── 分页器
```

### 弹窗

| 弹窗 | 说明 |
|------|------|
| 创建任务弹窗 | 客户信息 + 路线规划（含途径点、常用地址入口）+ 集装箱信息 + 业务类型 + 单证勾选 + 资源分配（创建并派车时）+ 调度备注 |
| 编辑任务弹窗 | 与创建弹窗结构一致，回填所有字段（仅待分配状态可用） |
| 分配弹窗 | 空闲司机列表 + 空闲车辆列表，单选确认 |
| 常用地址弹窗 | 地址列表 + 新增/删除/快速设为起运地或目的地 |

## 依赖关系

### 上游依赖
- **auth 模块**：用户认证和权限控制（已完成 ✅）
- **fleet 车队管理**：车辆和司机数据、车辆状态查询与更新 API（已完成 ✅）
  - 依赖 `GET /fleet/vehicles/{id}/availability` 检查车辆可用性
  - 依赖 `PUT /fleet/vehicles/{id}` 更新车辆状态
  - 依赖 `GET /fleet/vehicles` 获取空闲车辆列表（`status=idle`）
  - 依赖 `GET /fleet/drivers` 获取司机列表
  - **待补充**：fleet 需提供 `GET /fleet/drivers/{id}/availability` 接口，用于判断司机是否可分配（当前 fleet 的 Driver 模型无 status 字段，需通过关联车辆状态间接判断）
- **shared 模块**：公共组件和工具函数（已完成 ✅）

### 下游依赖
- **driver-app 司机端**：司机接收任务、更新任务状态（未开始）
