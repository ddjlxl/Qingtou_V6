## 一、功能概述

运营看板是 V6 的系统首页，让调度员和管理者一打开就能看到全局运营状况。页面以地图为主体，叠加浮动统计卡片和运力面板，3 秒内回答"今天整体怎么样、有没有超时的、车都在哪"。

---

## 二、用户故事

### US-001：查看核心指标
> 作为调度员，我希望一打开系统就能看到今日任务数、完成率、超时数和平均转运时间，以便快速判断今天运营状况。

### US-002：查看车辆位置
> 作为调度员，我希望在地图上看到所有车辆的实时位置和状态，以便直观了解车辆分布。

### US-003：地图与列表联动
> 作为调度员，我希望点击地图上的车辆标记能弹出信息，点击列表中的车辆能让地图飞到对应位置，以便快速定位特定车辆。

### US-004：查看任务状态分布
> 作为调度员，我希望看到各状态的任务数量（待分配/已分配/运输中/已完成/超时），以便掌握任务整体进度。

### US-005：数据自动刷新
> 作为调度员，我希望看板数据每 30 秒自动刷新，同时也能手动刷新，以便持续获取最新运营状况。

## 三、验收标准（AC）

### Happy Path

**AC-001**：核心指标数据与实际任务数据一致
- Given 调度员打开运营看板，When 页面加载完成，Then 左上角浮动显示 4 个指标卡片：
  - 今日任务数：今日创建的订单总数
  - 完成率：今日已完成订单数 / 今日非待分配订单数（assigned + transiting + completed + overdue），分母为 0 时显示 0%
  - 超时数：全局 status='overdue' 的订单数，>0 时红色高亮
  - 平均转运时间：今日已完成订单的平均转运分钟数，无数据时显示 "--"
- 指标数值与后端实际数据一致

**AC-002**：点击地图车辆标记弹出基本信息
- Given 地图上有车辆标记，When 点击某个车辆标记，Then 弹出信息窗口显示车牌号、司机姓名、当前状态
- 标记颜色按状态区分：空闲=绿色、运输中=蓝色、超时=红色

**AC-003**：地图与运力列表双向联动
- Given 运营看板已加载，When 点击右侧运力列表中的车辆，Then 地图自动飞到该车辆位置并高亮标记
- Given 运营看板已加载，When 点击地图上的车辆标记，Then 右侧列表对应行高亮并自动滚动到可见区域

**AC-004**：首页加载后地图正常显示
- Given 调度员登录系统，When 首页加载完成，Then 地图正常显示天地图瓦片（无天地图 Key 时降级为 OSM），有坐标的车辆标记出现在对应位置
- 无坐标的车辆不显示标记

### Edge & Error Cases

**AC-005**：无车辆数据时的显示
- Given 系统中无非停用车辆，When 运营看板加载完成，Then 地图正常显示（默认中心上海港区域），运力列表显示空状态提示"暂无车辆信息"

**AC-006**：搜索无结果时的显示
- Given 运力列表有车辆数据，When 搜索框输入的车牌号或司机姓名无匹配，Then 列表显示空状态提示"未找到匹配车辆"

**AC-007**：指标数据为空时的显示
- Given 今日无已完成订单，When 查看平均转运时间，Then 显示 "--"
- Given 今日无非待分配订单，When 查看完成率，Then 显示 0%

**AC-008**：车辆状态变更后标记颜色同步更新
- Given 地图上某车辆标记为绿色（空闲），When 该车辆状态变为运输中，Then 标记颜色自动变为蓝色，无需手动刷新

### Business Rules

**AC-009**：完成率计算规则
- 完成率 = 今日已完成订单数 / 今日非待分配订单数（assigned + transiting + completed + overdue）
- 分母为 0 时完成率返回 0%
- 完成率显示为百分比，保留 0 位小数

**AC-010**：数据刷新规则
- 看板数据每 30 秒自动刷新一次
- 提供手动刷新按钮，点击后立即刷新
- 刷新期间页面不闪烁，数据平滑更新

**AC-011**：权限控制
- 仅 admin 和 dispatcher 角色可访问运营看板
- driver 角色登录后跳转到司机端页面
- 未登录访问返回 401

**AC-012**：车辆标记渲染规则
- 仅显示非停用状态的车辆
- 有 lat/lng 坐标的车辆渲染标记，无坐标的不渲染
- 标记颜色：空闲(idle)=绿色 #67c23a、运输中(transiting)=蓝色 #409eff、超时(overdue)=红色 #f56c6c
- 首次加载车辆数据后，地图自动 fitBounds 到所有标记可见范围

## 四、业务流程

### 4.1 首页加载流程

```
调度员登录成功
  → 路由跳转到 /dashboard
    → 页面骨架渲染（LoadingSpinner）
      → 调用 GET /api/v1/dashboard
        → 返回 stats + status_counts + vehicles
          → 统计卡片渲染指标
          → 地图渲染车辆标记
          → 运力面板渲染列表 + 状态统计
    → 启动 30 秒自动刷新定时器
```

### 4.2 地图与列表联动流程

```
用户点击地图标记
  → MapArea emit('select-vehicle', vehicleId)
    → DashboardPage 更新 selectedVehicleId
      → FleetPanel 高亮对应行 + 滚动到可见
      → MapArea 保持当前标记高亮

用户点击列表项
  → FleetPanel emit('select-vehicle', vehicleId)
    → DashboardPage 更新 selectedVehicleId
      → MapArea flyTo 对应车辆位置 + 高亮标记
      → FleetPanel 高亮对应行
```

### 4.3 数据刷新流程

```
30 秒定时器触发 / 用户点击刷新按钮
  → store.fetchDashboard()
    → loading 状态更新（不显示全屏 LoadingSpinner，数据平滑替换）
      → 新数据替换旧数据
        → 统计卡片数值更新
        → 地图标记增量更新（新增/移除/移动/状态变更）
        → 运力列表更新
```

## 五、数据规则

| 规则 | 说明 |
|------|------|
| 今日任务数 | `COUNT(*) WHERE DATE(created_at) = today` |
| 完成率 | 今日已完成 / 今日非待分配（assigned + transiting + completed + overdue），0-1 保留原始精度 |
| 超时数 | 全局 `COUNT(*) WHERE status = 'overdue'`，非仅今日 |
| 平均转运时间 | `AVG(completed_at - assigned_at)` 分钟，仅今日已完成且有 assigned_at 的订单 |
| 车辆列表 | 所有非停用（disabled）状态的车辆，含关联司机信息 |
| 车辆坐标 | lat/lng 为 null 时不渲染地图标记 |
| 刷新间隔 | 30 秒自动刷新 |
| 权限 | admin / dispatcher 可访问 |

## 六、非功能需求

| 需求 | 说明 |
|------|------|
| 性能 | 首次加载 < 3 秒（含地图瓦片和车辆数据） |
| 兼容性 | 1920×1080 和 1366×768 分辨率下布局正常 |
| 地图瓦片 | 天地图优先（需环境变量 VITE_TIANDITU_KEY），无 Key 时降级 OSM |
| 内存 | 组件卸载时销毁地图实例，无内存泄漏 |

## 七、范围界定

### 本次做

- 运营看板页面（地图 + 统计 + 运力面板）
- 单一 API 接口 `GET /api/v1/dashboard`
- 30 秒自动刷新 + 手动刷新
- 地图与列表双向联动

### 本次不做

- 历史趋势图表（日/周/月维度的指标变化曲线）
- 车辆轨迹回放
- 地图上绘制区域/围栏
- 实时推送（WebSocket），当前用轮询替代
- 地图标记聚合（车辆密集区域自动聚合），当前车辆数量不需要
- 移动端适配，运营看板仅面向桌面端
