## 一、准备工作

### 联系之前先确认

- [ ] 你的**公司全称**（合同上的名字）
- [ ] 购买的**终端数量**（5 台）
- [ ] 第三方平台的**登录地址和账号**
- [ ] 能联系到的**人**：销售？技术？客服？
- [ ] 如果联系人是销售，第一句话就要让他转技术

### 找对人的路径

```
买设备的销售 → 让他拉厂商技术进群
        ↓
厂商技术 → 问 API 文档/技术对接方案
        ↓
如果厂商没技术能力 → 问销售"你们平台是买的哪家"
```

## 二、第一轮：问 API（最优先）

### 场景：第一次联系，不确定对方有没有开放能力

> **发送渠道**：微信/企业微信

---

> **话术 A（给销售/客服）**：
>
> 你好，我们公司之前在你们这里买了 5 台北斗定位终端，目前用你们平台看车辆位置，用得还可以。
>
> 现在我们想把这个定位数据**集成到我们自己内部的调度系统里面**，就是在地图上实时显示车辆位置。想问一下你们平台有没有**开放 API 接口**？我们可以通过接口拉取车辆经纬度数据。
>
> 如果有的话，方便拉一下你们技术人员沟通？或者直接发一份 API 文档给我也行。谢谢。

---

> **话术 B（直接给技术）**：
>
> 你好，我们公司买了你们 5 台北斗终端，目前在用你们的平台看车辆位置。
>
> 现在我们需要把定位数据**集成到自建系统**，想问几个技术问题：
>
> 1. 你们有没有 HTTP API 可以拉取车辆实时经纬度？
> 2. 如果有，API 的认证方式是什么？Token 还是账号密码？
> 3. 接口返回的数据格式是什么样的？有没有文档？
>
> 我们这边是 Python 技术栈，只要能调 HTTP 接口都能对接。麻烦发一份技术文档给我看看，谢谢。

## 三、第二轮：要数据转发（API 行不通时）

### 场景：对方说没有 API，或者 API 需要额外付费

> **话术**：
>
> 理解，那我想问一下，你们的终端支不支持**数据转发**？就是终端上报位置的时候，同时往我们自己服务器的 IP 和端口也发一份数据。
>
> 或者你们的云平台后台，能不能配置一个**数据推送地址**？把定位数据实时推送到我们指定的 HTTP 接口。
>
> 我们这边可以自己搭接收服务，只需要告诉我：
> - 用什么协议（TCP / HTTP / MQTT）？
> - 数据是什么格式（JSON / 二进制）？
> - 需要我们在终端后台配置什么参数（目标 IP、端口号）？
>
> 我们自己有服务器和开发能力，只要你们把数据发出来，我们就能接。

## 四、第三轮：强硬推进（对方不配合时）

### 场景：对方态度敷衍、说"做不到"、或者拖了很久不给回复

> **话术 A（搬出合规要求）**：
>
> 张工/李总，这个事我们也知道麻烦你们了，但确实是有硬性要求。
>
> **交通运输部《道路运输车辆动态监督管理办法》**规定，运输企业需要对营运车辆进行实时动态监控。我们现在上了 5 台北斗终端，如果数据不能接入我们自己的调度系统，后续监管检查的时候，我们拿不出车辆的实时轨迹数据，这个责任我们担不起。
>
> 而且我们公司后续业务扩展还会继续增加车辆和终端采购，数据能打通对我们长期合作是好事。
>
> 你看能不能帮我们想想办法？哪怕是给一个最基础的数据导出方式也行，我们自己处理。

---

> **话术 B（温和施压）**：
>
> 我们这个系统已经开发完了，现在就差定位数据接入这一块，卡了好久了。老板也在催进度。
>
> 你看能不能帮我确认一下，你们平台技术上到底能不能做到数据对接？如果确实做不到，我们也好早点想别的办法，不耽误双方时间。
>
> 如果能做到，需要什么条件（费用、开发周期），也麻烦跟我说一下，我去跟公司申请。

## 五、拿到 API 后问什么

### 如果对方给了 API 文档，按下面清单确认

| # | 确认事项 | 为什么重要 |
|---|---------|-----------|
| 1 | API 地址（Base URL）是什么？ | 代码里要写 |
| 2 | 认证方式：API Key / Token / 账号密码？ | 调用方式不同 |
| 3 | 有没有调用频率限制？每分钟/每小时限制多少次？ | 5 台车 10 秒查一次，每分钟查 30 次 |
| 4 | 返回的经纬度是什么坐标系？GCJ-02（高德/国测局）还是 WGS-84（GPS 原始）？ | 天地图用 **CGCS2000**（等价 WGS-84），如果是 GCJ-02 需要转换 |
| 5 | 返回数据包含哪些字段？ | 至少需要：经纬度、速度、时间 |
| 6 | 有没有测试环境或测试账号？ | 开发调试用 |
| 7 | 历史轨迹能查多久的？ | 决定轨迹回放功能的上限 |

### 坐标系转换（如果厂商返回的是 GCJ-02）

```python
# 高德/国测局坐标 (GCJ-02) → 天地图/国际标准 (WGS-84)
# 大部分北斗终端直接输出 WGS-84，不需要转
# 如果厂商平台返回的是 GCJ-02，才需要以下转换

import math

def gcj02_to_wgs84(lat: float, lng: float) -> tuple[float, float]:
    """GCJ-02 转 WGS-84（简版，精度 5 米内）"""
    a = 6378245.0
    ee = 0.00669342162296594323

    def transform_lat(x: float, y: float) -> float:
        ret = -100.0 + 2.0 * x + 3.0 * y + 0.2 * y * y + 0.1 * x * y + 0.2 * math.sqrt(abs(x))
        ret += (20.0 * math.sin(6.0 * x * math.pi) + 20.0 * math.sin(2.0 * x * math.pi)) * 2.0 / 3.0
        ret += (20.0 * math.sin(y * math.pi) + 40.0 * math.sin(y / 3.0 * math.pi)) * 2.0 / 3.0
        ret += (160.0 * math.sin(y / 12.0 * math.pi) + 320.0 * math.sin(y * math.pi / 30.0)) * 2.0 / 3.0
        return ret

    def transform_lng(x: float, y: float) -> float:
        ret = 300.0 + x + 2.0 * y + 0.1 * x * x + 0.1 * x * y + 0.1 * math.sqrt(abs(x))
        ret += (20.0 * math.sin(6.0 * x * math.pi) + 20.0 * math.sin(2.0 * x * math.pi)) * 2.0 / 3.0
        ret += (20.0 * math.sin(x * math.pi) + 40.0 * math.sin(x / 3.0 * math.pi)) * 2.0 / 3.0
        ret += (150.0 * math.sin(x / 12.0 * math.pi) + 300.0 * math.sin(x / 30.0 * math.pi)) * 2.0 / 3.0
        return ret

    dlat = transform_lat(lng - 105.0, lat - 35.0)
    dlng = transform_lng(lng - 105.0, lat - 35.0)
    radlat = lat / 180.0 * math.pi
    magic = math.sin(radlat)
    magic = 1 - ee * magic * magic
    sqrtmagic = math.sqrt(magic)
    dlat = (dlat * 180.0) / ((a * (1 - ee)) / (magic * sqrtmagic) * math.pi)
    dlng = (dlng * 180.0) / (a / sqrtmagic * math.cos(radlat) * math.pi)
    return (lat - dlat, lng - dlng)
```

## 六、如果对方说"要加钱"

### 可能的情况和应对

| 对方说法 | 意思 | 应对 |
|---------|------|------|
| "API 接口需要额外购买" | 平台有 API 但收费 | 问价格。一般几百到几千一年，5 台车数据量不大，通常不贵 |
| "需要升级套餐" | 低档套餐不含 API | 问升级费用和包含的服务 |
| "需要定制开发" | 他们没有现成的 API | 问定制费用和时间。如果太贵，考虑方案 B 或换终端 |

> **话术**：
>
> 大概费用是多少？我们了解清楚好跟公司申请。
>
> 另外我想确认一下，开通 API 之后，调用有没有限制？我们 5 台车大概 10 秒查一次位置，这个频率能支持吗？
>
> 还有 API 开通后多久能好用？我们这边比较急。

## 七、常见 QA 应对

### Q1："你们要 API 做什么用？"

> 我们公司自己在开发一个内部的调度管理系统，想把车辆定位集成进去，这样调度员在一个系统里就能看到所有信息，不用在我们系统和你这边来回切换。

### Q2："你们的服务器在哪？需要公网 IP 吗？"

> 我们需要一个公网 IP 或者云服务器来接收数据。如果是 API 拉取方式，我们服务器能访问外网就可以；如果是 TCP 转发方式，需要一台有公网 IP 的服务器。这个我们可以自己准备。

### Q3："终端不支持转发怎么办？"

> 那终端有没有**SDK 或二次开发接口**？或者终端能否通过**RS232/RS485 串口**输出定位数据？如果都不行的话，我们可能要考虑换一批支持 TCP 上报的终端了。

### Q4："你们有多少辆车？"

> 目前自有车 5 台，都装了你们终端。后续业务扩展还会增加。这次先把 5 台打通，后面加车就是重复操作。

### Q5："你们用的是什么地图？天地图还是高德？"

> 我们用天地图（CGCS2000/WGS-84 坐标系）。如果你们平台返回的是高德坐标（GCJ-02），告诉我们，我们自己可以转。

### Q6："为什么要实时数据？30 秒查一次不行吗？"

> 调度员需要看到车辆动态位置来判断车辆在不在作业点、有没有在路上。实时更新（10-30 秒一次）就够用了，不需要秒级刷新。

### Q7："数据量会不会太大？"

> 不用担心数据量的问题。我们 5 台车 30 秒上报一次，一天大概 1.4 万条数据，存一年也就 200MB 左右，我们的数据库完全能承受。
