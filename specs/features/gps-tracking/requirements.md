# 车载北斗定位接入 — 需求文档

> **版本**：v1.0
> **创建日期**：2026-05-29
> **状态**：待确认
> **依赖**：dashboard 模块（地图展示）、fleet 模块（车辆/司机数据）

---

## 一、功能概述

将公司 5 台自有车辆已安装的车载北斗定位终端数据接入 V6 系统，实现 Dashboard 地图上车辆位置的**实时动态更新**，取代当前 seed 脚本写死静态坐标的方案。

---

## 二、现状与目标

| 维度 | 当前状态 | 目标状态 |
|------|---------|---------|
| 车辆位置 | seed 脚本硬编码写死坐标，地图上车辆不会移动 | 每 10-30 秒刷新一次，车辆在地图上实时移动 |
| 数据来源 | 无，全靠手动 seed | 北斗终端自动上报经纬度 |
| 坐标精度 | 人工估算，不准确 | 终端实际 GPS/北斗定位，精度 5-10 米 |
| 外协车辆 | 无坐标数据 | 外协车无北斗时不显示，有则显示 |

---

## 三、用户故事

### US-001：实时查看车辆位置
> 作为调度员，我希望在 Dashboard 地图上看到车辆的**实时位置**，车辆移动时地图上的标记也随之移动，以便随时掌握车辆动态。

### US-002：查看车辆状态关联位置
> 作为调度员，我希望看到车辆当前状态（空闲/运输中/超时）与位置信息的关联，以便判断车辆是否在作业点位、是否偏离路线。

### US-003：查看上次定位时间
> 作为调度员，我希望看到车辆**最近一次 GPS 上报时间**，以便判断车辆是否在线、数据是否过期。

## 四、验收标准（AC）

### Happy Path

**AC-001**：北斗终端数据成功接入系统
- Given 车辆已安装北斗终端并正常开机，When 终端上报位置数据，Then 系统后台正确接收到经纬度、速度、方向、时间信息
- 数据经过解析后存入数据库

**AC-002**：Dashboard 地图显示实时位置
- Given 调度员打开 Dashboard 页面，When 车辆位置有更新，Then 地图上对应车辆的 Marker 自动移动到新位置（增量更新，不闪烁、不抖动）
- 位置刷新间隔不超过 30 秒

**AC-003**：车辆离线状态可识别
- Given 某车辆北斗终端断网/断电超过 5 分钟，When Dashboard 页面加载或刷新，Then 该车辆 Marker 显示为灰色半透明，表示"数据过期"
- 鼠标悬停或点击 Popup 显示"最后定位时间：X 分钟前"

**AC-004**：不因定位数据影响其他业务
- Given 定位服务运行正常，When 车辆其他业务数据（订单、状态）发生变化，Then 不受定位数据影响，业务功能正常

**AC-005**：系统在无定位数据时正常降级
- Given 北斗终端离线或厂商 API 不可用，When Dashboard 页面加载，Then 地图正常显示，车辆显示为"数据离线"状态，不报错、不崩溃

### Edge & Error Cases

**AC-006**：终端数据异常处理
- Given 终端上报的经纬度为 (0,0) 或明显异常值（超出中国范围），Then 系统自动过滤该条数据，不更新车辆位置
- Given 终端上报速度超过 200km/h，Then 系统记录日志但正常显示位置

**AC-007**：数据存储容量控制
- Given 5 台车持续上报数据（10 秒/次），When 连续运行 30 天，Then 数据库 GPS 数据量不超过 1GB
- 轨迹数据自动按时间分区，超过 90 天的数据自动清理
