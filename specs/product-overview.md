> **版本**：v1.0  
> **创建日期**：2026-05-03

## 一、产品愿景

- **产品名称**：青投供应链 V6
- **一句话描述**：面向中小物流团队的运输调度管理系统，让调度员高效分配任务、司机便捷接单执行，管理者一目了然掌握全局。
- **核心价值**：V4 功能完整但代码混乱、BUG 多、不敢改。V6 用严格类型安全、清晰分层架构、完善测试覆盖，让系统真正能稳定用于生产。与 V4 相比，V6 不是功能堆砌，而是**可靠性的质变**——同样的功能，但调度员不再担心点错按钮系统崩溃，司机不再遇到状态卡死，管理者不再看到数据对不上。
- **目标用户**：
  - 调度员（3-5 人，电脑浏览器）—— 日常在办公室创建和分配运输任务
  - 司机（10-30 人，手机浏览器）—— 在运输途中接单、更新状态
  - 管理员（1-2 人）—— 管理系统配置和用户权限

## 二、核心板块

### 板块 1：运营看板（首页）
- **职责**：作为系统首页，让调度员和管理者一打开就能看到全局运营状况
- **关键功能**：
  - 全局核心指标（今日任务数、完成率、超时数、平均转运时间）
  - 天地图地图 + 车辆实时定位标记
  - 地图与运力列表双向联动（点地图上的车 → 右侧列表高亮，点列表 → 地图飞到该车）
  - 任务状态概览（待分配 / 进行中 / 超时数量）
  - 30 秒自动刷新 + 手动刷新
- **用户价值**：调度员上班第一件事就是打开这个页面，3 秒内了解"今天整体怎么样、有没有超时的、车都在哪"

### 板块 2：调度中心
- **职责**：调度员的核心工作台，管理运输任务的全生命周期
- **关键功能**：
  - 创建任务（客户信息、起运地/目的地、箱号箱型封号、业务类型）
  - 任务列表（搜索、日期筛选、状态筛选、分页）
  - 分配司机和车辆（自动检查可用性）
  - 任务状态流转（待分配 → 已分配 → 运输中 → 已完成 / 已超时）
  - 智能调度引擎（距离优先、司机评分、时效权重可配置）
  - 司机智能推荐
  - 常用地址快速选择
  - 批量操作（删除、导出）
  - **OCR 集装箱号码识别**（拍照或上传图片自动识别箱号，减少手动输入错误）
- **用户价值**：调度员 80% 的工作在这个页面完成，从创建到分配到跟踪，一条龙不切换页面。OCR 识别让箱号输入从 30 秒手动打字变成 3 秒拍照识别

### 板块 3：车队管理
- **职责**：管理车辆和司机的档案、状态、证照
- **关键功能**：
  - 车辆增删改查（车牌、归属性质、关联司机）
  - 司机信息管理
  - 车辆状态跟踪（空闲 / 运输中 / 超时）
  - 证照管理（到期预警）
  - 运输流水记录
  - 车队统计（证照预警数、今日里程、本月里程、总任务数）
- **用户价值**：车队长或调度员定期维护车辆档案，证照快到期了系统自动提醒，不用人工记

### 板块 4：仓库管理
- **职责**：管理仓库库位和集装箱存放
- **关键功能**：
  - 仓库信息管理
  - 库位可视化网格布局（颜色区分空位/重箱/空箱）
  - 入库/出库操作
  - 库存查询和筛选
  - 集装箱在库位之间移动
  - 批量导入（文件 + 手动输入）
  - 仓库统计（总库位、已用、利用率）
- **用户价值**：仓库管理员一眼看到哪些库位空着、哪些箱停了多久，操作直观不靠脑子记

### 板块 5：系统设置
- **职责**：系统级配置和安全管理
- **关键功能**：
  - 用户管理（增删改查、角色分配）
  - 系统参数配置
  - 操作日志查看
  - 调度参数设置
- **用户价值**：管理员独立管理账号和系统配置，不影响调度员日常操作

### 板块 6：帮助中心（V6 新增）
- **职责**：为调度员和司机提供图文操作教程
- **关键功能**：
  - 按模块分类的图文教程
  - 后台可编辑（管理员可新增、编辑、删除教程，无需改代码部署）
- **用户价值**：新来的调度员或司机不用找人教，自己看教程就能上手

### 板块 7：司机端（独立页面）
- **职责**：司机在手机上使用的轻量页面，完成接单和状态上报
- **关键功能**：
  - 司机登录
  - 任务大厅（浏览可接任务）
  - 我的任务（已分配任务列表）
  - 任务详情（查看详情、更新状态）
  - 历史记录
  - 回填页
  - 新任务消息通知（调度员分配后自动提示）
- **用户价值**：司机在运输途中用手机打开网页就能操作，不需要装 App，有新任务自动提醒不用反复刷新

## 三、验收标准

### 功能验收

**运营看板**
- [ ] 首页加载后，地图正常显示天地图瓦片，车辆标记出现在对应坐标位置
- [ ] 点击地图上的车辆标记，弹出车辆基本信息（车牌、司机、状态）
- [ ] 点击右侧运力列表中的司机，地图自动飞到该车辆位置并高亮
- [ ] 核心指标数据与实际任务数据一致（任务数、完成率、超时数）

**调度中心**
- [ ] 填写完整表单后，任务创建成功并出现在列表中，任务编号自动生成
- [ ] 起运地和目的地相同时，系统拒绝创建并给出明确提示
- [ ] 分配司机时，如果司机已有进行中任务，系统提示"该司机当前不可用"
- [ ] 任务状态只能按合法路径流转（待分配→已分配→运输中→已完成/已超时），非法转换被拒绝
- [ ] 常用地址保存后可快速选择，无需重复输入
- [ ] OCR 拍照识别箱号后，识别结果自动填入箱号字段，识别准确率 > 90%

**车队管理**
- [ ] 新增车辆后出现在列表中，编辑后信息更新，删除后有确认提示
- [ ] 证照即将到期时，统计卡片显示预警数量
- [ ] 车辆状态随任务分配自动更新（分配后变为运输中，完成后恢复空闲）

**仓库管理**
- [ ] 库位网格正确渲染，空位/重箱/空箱颜色区分明显
- [ ] 点击库位显示详情（箱号、箱型、停留天数）
- [ ] 集装箱从一个库位移动到另一个库位后，两个库位状态正确更新
- [ ] 批量导入数据后，库位状态正确更新

**系统设置**
- [ ] 新增用户后可用新账号登录
- [ ] 操作日志记录关键操作（创建任务、分配司机、状态变更）

**帮助中心**
- [ ] 管理员可在后台新增、编辑、删除教程
- [ ] 教程内容修改后前端立即生效，无需重新部署

**司机端**
- [ ] 司机登录后看到任务大厅，可浏览待分配任务
- [ ] 调度员分配任务后，司机端收到新任务提示
- [ ] 司机可更新任务状态（开始运输、完成运输），状态变更同步到调度端

### 性能验收

- [ ] 运营看板首次加载 < 3 秒（含地图瓦片和车辆数据）
- [ ] 任务列表翻页响应 < 1 秒

### 体验验收

- [ ] 调度端在 1920×1080 和 1366×768 分辨率下布局正常
- [ ] 司机端在主流手机屏幕（375-414px 宽）下布局正常，按钮可点击区域足够大
- [ ] 表单校验失败时，错误提示明确指向具体字段

## 四、版本规划

### MVP（最小可行产品）
所有"必须有"功能一次性交付，因为这是重写项目而非从零开发——七大模块缺一不可，少了任何一个都会导致业务流程断裂。

### V1.0（完整版）
MVP 即 V1.0，包含全部七大模块 + OCR 箱号识别 + 消息通知 + 状态简化。

### V2.0（增强版，未来考虑）
- WebSocket 实时推送通知
- 数据导出增强（更多格式、自定义字段）
- 移动端 PWA 支持（司机端可添加到手机桌面）

## 六、约束与风险

- **技术约束**：零容忍 `any` 类型、零容忍 `console.log` 遗留、文件 ≤ 300 行、函数 ≤ 50 行
- **时间约束**：质量优先，不赶工期
- **预算约束**：无特殊限制
- **主要风险**：
  - **V4 功能遗漏风险**：重写时可能遗漏 V4 的隐含功能。缓解措施——开发前逐模块对照 V4 代码确认功能清单
  - **业务规则理解偏差风险**：V4 代码混乱导致业务规则难以提取。缓解措施——遇到不确定的规则时优先与用户确认，不猜测
  - **司机端兼容性风险**：独立 HTML 页面需在多种手机浏览器上测试。缓解措施——覆盖 iOS Safari、Android Chrome、微信内置浏览器
  - **OCR 识别精度风险**：不同光照、角度下识别率可能下降。缓解措施——识别结果允许手动修正，提供重新拍照入口
