# 项目需求澄清

> **版本**：v1.0  
> **创建日期**：2026-05-03  
> **来源**：基于 V4 代码分析和引导式需求对话  

---

## 项目概述

- **项目名称**：青投供应链 V6
- **一句话描述**：基于 V4 的物流运输调度系统，功能不变，用高质量代码重写，目标是生产可用率 > 90%
- **核心价值**：V4 功能完整但代码混乱、BUG 多、不敢改。V6 用严格类型安全、清晰分层架构、完善测试覆盖，让系统真正能稳定用于生产

---

## 目标用户

- **主要用户**：
  - 调度员（电脑浏览器）—— 创建运输任务、分配司机车辆、跟踪任务进度
  - 司机（手机浏览器）—— 查看任务大厅、接单、更新任务状态、查看历史
  - 管理员 —— 系统配置、用户管理、查看数据统计
- **使用场景**：日常物流运输调度管理，调度员在办公室用电脑操作，司机在途中用手机操作
- **用户规模预估**：小团队（预计调度员 3-5 人，司机 10-30 人）

---

## 核心功能

### 必须有（MVP）

#### 1. 用户认证模块
- 登录/登出
- 角色权限控制（调度员、司机、管理员）
- 登录状态持久化

#### 2. 运营看板（首页）
- 全局核心指标统计（今日任务数、完成率、超时数、平均转运时间等）
- **天地图集成 + 车辆实时定位**（Leaflet，支持街道/卫星图切换）
- 车辆标记点击查看详情，地图与运力列表双向联动
- 任务状态概览卡片（待分配/进行中/超时统计）
- 运力实时状态列表
- 智能刷新（30 秒自动 + 手动刷新按钮）

#### 3. 调度中心
- 创建运输任务（客户信息、起运地/目的地、箱号箱型封号、业务类型、备注）
- 任务列表（搜索、日期范围筛选、状态筛选、分页）
- 分配司机和车辆（检查司机和车辆可用性）
- 任务状态流转（4 种状态：待分配 → 已分配 → 运输中 → 已完成 / 超时）
- 智能调度引擎（距离优先、司机评分、时效权重可配置）
- 司机推荐（基于策略模式）
- 批量操作（多选删除、导出）
- **常用地址**（保存频繁使用的起运地/目的地，快速选择）
- **OCR 集装箱号码识别**（拍照或上传图片自动识别箱号，减少手动输入错误）

#### 4. 车队管理
- 车辆信息管理（新增、编辑、删除）
- 司机信息管理
- 车辆状态跟踪（空闲 / 运输中 / 超时）
- 证照管理（证照到期预警）
- 运输流水记录
- 车队专属统计（证照预警、今日里程、本月里程、总任务数）

#### 5. 仓库管理
- 仓库信息管理
- 库位可视化布局（网格图，不同颜色表示空/重/空箱状态）
- 入库/出库操作
- 库存查询和筛选
- 集装箱移动（从一个库位移动到另一个库位）
- 批量导入（文件导入 + 手动输入）
- 仓库统计（总库位、已用、利用率）

#### 6. 系统设置
- 用户管理（增删改查）
- 系统参数配置
- 操作日志查看
- 调度参数设置

#### 7. 帮助中心（新增）
- 图文操作手册，分模块展示
- 后台可编辑管理（管理员可新增、编辑、删除教程内容，无需改代码部署）

#### 8. 司机端（独立页面）
- 司机登录
- 任务大厅（浏览可接任务）
- 我的任务（已分配任务列表）
- 任务详情（查看和更新任务状态）
- 历史记录
- 回填页

### 应该有（V6 新增改进）

- **消息通知**：调度员分配任务后，司机端页面自动提示新任务（无需刷新），列表中标记新任务
- **任务状态简化**：从 V4 的 9 种状态简化为 5 种（待分配 → 已分配 → 运输中 → 已完成 / 已超时），减少混淆和 BUG
- **车辆状态简化**：从 V4 的 4 种状态简化为 3 种（空闲 / 运输中 / 超时），去除"异常"状态

### 可以有（未来考虑）

- 推送通知（WebSocket 实时推送）
- 数据导出增强（更多格式、自定义字段）
- 移动端 PWA 支持

## 关键约束

- **平台**：Web 端 + 移动端响应式适配（一套 Vue 代码覆盖调度员电脑端和司机手机浏览器）
- **司机端**：独立 HTML 页面（保持简单、加载快、易维护），与调度端共享同一后端 API
- **时间要求**：正常（质量优先，不赶工期）
- **预算限制**：无特殊限制
- **特殊要求**：
  - 零容忍：禁止 `any` 类型、禁止 `console.log` 遗留、文件不超过 300 行、函数不超过 50 行
  - 必须通过 TypeScript 严格模式检查
  - 核心功能测试覆盖率 > 80%
