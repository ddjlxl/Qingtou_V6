# V6项目上下文协议

> **版本**：v2.0  
> **用途**：强制AI在执行前读取项目文档，建立完整认知  
> **作用范围**：所有Skills执行时自动加载

---

## 核心原则

AI最大的问题是什么？**没有记忆。**

项目上下文协议的规则很简单：**每个Skill执行前，AI必须先读取项目文档，建立对项目的完整认知。**

文档在，记忆就在。

---

## 强制读取规则

### 必读文档（优先级：高）

每个会话开始时，必须按顺序读取以下文档：

#### 0. 进度追踪（最优先）
**路径**：`PROGRESS.md`
**内容**：当前进度、已完成任务、下一步建议
**作用**：知道做到哪了、接下来该做什么

#### 1. 需求文档
**路径**：`specs/requirements-clarification.md`
**内容**：功能需求、业务规则、验收标准
**作用**：了解要做什么

#### 2. 产品概述
**路径**：`specs/product-overview.md`
**内容**：产品愿景、核心模块、成功指标
**作用**：了解产品全貌

#### 3. 技术栈
**路径**：`specs/tech-stack.md`
**内容**：前端/后端技术选型及理由
**作用**：了解用什么技术

#### 4. 项目结构
**路径**：`specs/project-structure.md`
**内容**：目录结构、模块组织方式
**作用**：了解代码放哪

#### 5. 开发规范
**路径**：`specs/development-standards.md`
**内容**：TypeScript规范、组件规范、API规范、Git规范
**作用**：了解怎么写代码

#### 6. 开发路线图
**路径**：`specs/development-roadmap.md`
**内容**：里程碑、Phase划分、依赖关系
**作用**：了解开发顺序

## 上下文验证

### 读取完成后的输出格式

读取完成后，AI必须输出以下验证信息：

```
✅ 已加载项目上下文：

进度追踪：
- PROGRESS.md：已读取（当前阶段：XXX，下一步：XXX）

核心文档：
- 需求文档：已读取（X个功能模块，X条业务规则）
- 产品概述：已读取（X个核心模块）
- 技术栈：已读取（前端：XXX，后端：XXX）
- 项目结构：已读取（X个业务模块）
- 开发规范：已读取（X条规则）
- 路线图：已读取（X个里程碑）

项目认知：
- 技术栈：Vue 3 + TypeScript + Element Plus + FastAPI + SQLite
- 核心功能：任务管理、车辆管理、仓库管理、运营看板
- 开发约束：无any类型、无console.log、文件不超过300行
```

### 交叉验证

读取 PROGRESS.md 后，必须与实际代码目录交叉验证：

- 扫描 `apps/frontend/src/modules/` 下已有模块
- 扫描 `apps/server/app/` 下已有代码
- 如果 PROGRESS.md 与实际代码状态不一致，以实际代码为准，并主动报告漂移

## 上下文使用规则

### 1. 技术栈一致性
- 必须使用文档中定义的技术栈
- 不得擅自引入新技术或框架
- 如需修改技术栈，必须先更新文档

### 2. 规范遵循
- 必须遵循文档中定义的开发规范
- 代码风格必须一致
- 命名必须符合规范

### 3. 避免重复错误
- 必须参考 ai-constraints.md 中的 V4 教训
- 不得重复 V4 的错误

---

## 上下文更新

### 何时更新
- 需求变更时
- 技术方案调整时
- 发现新的最佳实践时

### 如何更新
1. 先更新文档
2. 再更新代码
3. 记录变更原因

---

## 强制执行

### 自动加载
每个会话开始时，系统会自动加载项目上下文。

### 验证失败处理
如果读取失败或验证不通过：
```
❌ 项目上下文加载失败

缺失文档：
- [文档名称]：文件不存在

请先创建缺失的文档，或检查文件路径是否正确。
```
