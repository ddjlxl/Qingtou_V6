# 垂直切片策略指南

> 本文档为 `task-planning` Skill 的参考指南，提供任务拆分的策略和示例。

---

## 什么是垂直切片

**垂直切片**：一个任务穿透所有技术层（数据库 → API → Store → UI），实现一个完整的用户可感知的功能。

**水平切片**（不推荐）：按技术层拆分——先做所有数据库、再做所有 API、再做所有 UI。这种拆分方式导致每个阶段结束时没有可交付的功能。

---

## 垂直切片示例

### 功能：用户管理

#### ❌ 水平切片（不推荐）
```
任务 1：设计所有数据库表（用户表、角色表、权限表）
任务 2：实现所有 API 接口（CRUD 用户、CRUD 角色、CRUD 权限）
任务 3：实现所有 Store（用户 Store、角色 Store、权限 Store）
任务 4：实现所有 UI 页面（用户列表、角色列表、权限列表）
```

问题：任务 1 完成后，没有任何可用的功能。

#### ✅ 垂直切片（推荐）
```
任务 1：用户列表查看
  - 数据库：users 表
  - API：GET /api/v1/users
  - Store：useUserStore.fetchUsers()
  - UI：UserList.vue（表格 + 分页 + 搜索）
  - 测试：API 测试 + 组件测试

任务 2：用户创建
  - 数据库：users 表（已有）
  - API：POST /api/v1/users
  - Store：useUserStore.createUser()
  - UI：UserCreateDialog.vue（表单 + 验证）
  - 测试：API 测试 + 组件测试

任务 3：用户编辑
  - 数据库：users 表（已有）
  - API：PUT /api/v1/users/:id
  - Store：useUserStore.updateUser()
  - UI：UserEditDialog.vue（表单 + 预填数据）
  - 测试：API 测试 + 组件测试

任务 4：用户删除
  - 数据库：users 表（已有）
  - API：DELETE /api/v1/users/:id
  - Store：useUserStore.deleteUser()
  - UI：确认对话框 + 列表刷新
  - 测试：API 测试 + 组件测试
```

每个任务完成后，用户都能看到一个完整的功能。

## 切片粒度判断

| 功能复杂度 | 切片粒度 | 示例 |
|-----------|---------|------|
| 简单 CRUD | 每个操作一个任务 | 创建用户、编辑用户、删除用户 |
| 中等功能 | 每个用户场景一个任务 | 用户注册流程、用户登录流程 |
| 复杂功能 | 每个子功能一个任务 | 订单创建、订单支付、订单退款 |

---

## 任务依赖管理

```
任务 1：用户列表查看（无依赖，先做）
    ↓
任务 2：用户创建（依赖任务 1 的 Store 结构）
    ↓
任务 3：用户编辑（依赖任务 1 的 Store 结构）
    ↓
任务 4：用户删除（依赖任务 1 的 Store 结构）
```

原则：
- 先做"查看"类任务（建立数据流）
- 再做"操作"类任务（基于已有数据流）
- 最后做"管理"类任务（批量操作、权限等）
