# 代码审查检查清单

> 本文档为 `code-review` Skill 的参考检查清单，审查时逐项确认。

---

## 一、TDD 合规性

- [ ] 有对应的测试文件（`.test.ts` / `test_*.py`）
- [ ] 测试在实现代码之前提交（通过 git log 验证）
- [ ] 测试覆盖了所有 AC（对照 `tasks.md`）
- [ ] 测试中引用了 AC 编号
- [ ] 测试有明确的输入和预期输出
- [ ] 测试有断言（不是只调用不验证）

## 二、类型安全

- [ ] 无 `any` 类型（搜索 `: any` 和 `as any`）
- [ ] 所有函数参数有显式类型
- [ ] 所有函数返回值有显式类型
- [ ] Props 使用 TypeScript 接口定义
- [ ] Emits 使用 TypeScript 接口定义
- [ ] API 请求/响应类型已定义
- [ ] 无 `@ts-ignore` 或 `@ts-nocheck`

## 三、代码质量

- [ ] 无 `console.log` / `print` / `debugger`
- [ ] 无注释掉的代码块
- [ ] 无硬编码的魔法数字（提取为常量）
- [ ] 无硬编码的字符串（提取为常量或配置）
- [ ] 异步操作有 try-catch 错误处理
- [ ] 错误信息对用户友好
- [ ] 无未使用的导入
- [ ] 无未使用的变量

## 四、组件规范

- [ ] 处理了 loading 状态（骨架屏/加载动画）
- [ ] 处理了 empty 状态（空状态提示）
- [ ] 处理了 error 状态（错误提示 + 重试）
- [ ] 样式使用 `<style scoped>`
- [ ] 无内联样式（动态值除外）
- [ ] 组件命名使用 PascalCase
- [ ] Props 有默认值（`withDefaults`）

## 五、结构规范

- [ ] 文件不超过 300 行
- [ ] 函数不超过 50 行
- [ ] 单一职责：一个文件/函数只做一件事
- [ ] 导入按类型分组（第三方 → 项目内部 → 相对路径）
- [ ] 无循环依赖

## 六、安全性

- [ ] 用户输入有验证（前端 + 后端）
- [ ] 无 SQL 注入风险（使用参数化查询）
- [ ] 无 XSS 风险（使用 v-text / 转义）
- [ ] 无暴露的密钥、Token、API Key
- [ ] 敏感操作有认证检查
- [ ] 无在前端硬编码的后端 URL（使用环境变量）

## 七、业务逻辑

- [ ] 实现符合技术方案设计
- [ ] 边界情况已处理（空值、超长、特殊字符）
- [ ] 异常流程有正确处理
- [ ] 无引入与需求不符的行为
- [ ] 数据验证规则与 AC 一致

## 八、性能

- [ ] 列表渲染使用 `v-for` + `:key`
- [ ] 大列表考虑虚拟滚动
- [ ] 图片使用懒加载
- [ ] 无不必要的重复渲染
- [ ] 计算属性无副作用
