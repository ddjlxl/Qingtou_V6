# AI 协作协议

> 本文档定义 AI 在本项目中生成代码时必须遵守的契约。每个 Skill 执行时自动加载。

---

## 零容忍规则

以下规则任何情况下不可违反：

### 1. 类型安全
- **禁止使用 `any` 类型**。使用具体类型、泛型或 `unknown` + 类型守卫。
- 所有函数参数和返回值必须有显式类型。
- Props、Emits、API 请求/响应必须有 TypeScript 接口定义。

### 2. 调试代码
- **禁止遗留 `console.log`、`print`、`debugger` 等调试代码**。
- 使用项目日志系统（`logger.debug()`、`logger.error()` 等）代替。
- 开发环境需要临时调试时，使用条件编译：`if (import.meta.env.DEV) { ... }`

### 3. 文件大小
- **单个文件不超过 300 行**。超过则必须拆分为多个模块。
- **单个函数不超过 50 行**。超过则提取子函数。

### 4. 错误处理
- 所有异步操作必须有 `try-catch` 错误处理。
- 错误信息必须对用户友好，不能直接暴露技术细节。
- 后端 API 必须返回正确的 HTTP 状态码和错误信息。

### 5. 状态覆盖
- 所有数据展示组件必须处理三种状态：**loading**、**empty**、**error**。
- 不能假设数据一定存在且加载成功。

### 6. 样式隔离
- 所有组件样式必须使用 `<style scoped>`。
- 禁止使用内联样式（除非动态计算的值）。
- 遵循项目 CSS 变量和设计令牌体系。

### 7. Mock 分离
- **禁止在 Store 或业务逻辑中混入 Mock 数据**。
- Mock 数据统一放在 `__mocks__/` 目录。
- 通过环境变量控制是否使用 Mock：`VITE_USE_MOCK=true`。

### 8. 代码注释
- 公共函数、复杂逻辑必须有 JSDoc 注释。
- 注释说明"为什么这样做"，而不是"做了什么"（代码本身已经说明了做什么）。

## 代码生成检查清单

每次生成代码前，AI 必须自检：

- [ ] 没有使用 `any` 类型
- [ ] 没有 `console.log` 调试代码
- [ ] 文件不超过 300 行
- [ ] 函数不超过 50 行
- [ ] 有完整的错误处理
- [ ] 组件处理了 loading/empty/error 状态
- [ ] 样式使用了 `scoped`
- [ ] 没有混入 Mock 数据
- [ ] 公共函数有 JSDoc 注释
