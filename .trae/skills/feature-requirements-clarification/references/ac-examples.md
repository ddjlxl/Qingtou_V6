# 验收标准（AC）编写示例

> 本文档为 `feature-requirements-clarification` Skill 的参考附录，提供各类 AC 的编写示例。

---

## AC 编写原则

好的 AC 必须满足 **SMART** 原则：
- **S**pecific（具体）：明确描述什么行为
- **M**easurable（可衡量）：可以验证是否完成
- **A**chievable（可实现）：技术上可行
- **R**elevant（相关）：与功能需求直接相关
- **T**estable（可测试）：能写自动化测试验证

---

## 示例 1：用户登录

### ❌ 不好的 AC
```
AC-001: 用户可以登录
AC-002: 登录要安全
AC-003: 体验要好
```

### ✅ 好的 AC
```
AC-001: 用户输入正确的手机号和密码后，点击登录按钮，系统验证通过后跳转到首页
AC-002: 用户输入错误的手机号或密码，系统显示"手机号或密码错误"，不跳转
AC-003: 用户连续 5 次输入错误密码，账号锁定 30 分钟，显示"账号已锁定，请30分钟后重试"
AC-004: 用户未输入手机号点击登录，手机号输入框下方显示"请输入手机号"
AC-005: 用户未输入密码点击登录，密码输入框下方显示"请输入密码"
AC-006: 登录成功后，系统生成 JWT Token，有效期 7 天
AC-007: Token 过期后，用户访问需要登录的页面时，自动跳转到登录页
```

## 示例 2：订单列表

### ❌ 不好的 AC
```
AC-001: 显示订单列表
AC-002: 可以筛选
AC-003: 支持分页
```

### ✅ 好的 AC
```
AC-001: 进入订单列表页，默认显示最近 30 天的订单，按创建时间倒序排列
AC-002: 订单列表每页显示 20 条，底部显示分页器，显示总页数和总条数
AC-003: 用户可以通过订单状态（待付款/已付款/已发货/已完成/已取消）筛选订单
AC-004: 用户可以通过日期范围筛选订单，默认最近 30 天
AC-005: 用户可以通过订单号或商品名称搜索订单
AC-006: 订单列表为空时，显示"暂无订单"空状态，并提供"去逛逛"按钮跳转到首页
AC-007: 订单列表加载中时，显示骨架屏（6 个卡片占位）
AC-008: 订单列表加载失败时，显示"加载失败，请重试"，并提供重试按钮
AC-009: 每个订单卡片显示：订单号、下单时间、商品缩略图、商品名称、数量、金额、状态
AC-010: 点击订单卡片，跳转到订单详情页
```

## 示例 3：文件上传

### ❌ 不好的 AC
```
AC-001: 可以上传文件
AC-002: 支持多种格式
AC-003: 文件不能太大
```

### ✅ 好的 AC
```
AC-001: 用户点击上传按钮，弹出文件选择器，支持选择单个文件
AC-002: 支持上传的文件格式：jpg、png、gif、pdf、doc、docx、xls、xlsx
AC-003: 单个文件大小不超过 10MB，超过时显示"文件大小不能超过10MB"
AC-004: 上传过程中显示进度条，显示百分比和上传速度
AC-005: 上传成功后，显示文件缩略图（图片）或文件图标（文档），并显示文件名和大小
AC-006: 上传失败时，显示"上传失败，请重试"，并提供重试按钮
AC-007: 用户可以取消正在上传的文件
AC-008: 用户可以删除已上传的文件，删除前弹出确认对话框
AC-009: 同时最多上传 3 个文件，超过时显示"最多同时上传3个文件"
```

## AC 编号规范

```
AC-XXX：验收标准编号
格式：AC-001, AC-002, AC-003...

在测试代码中引用：
```python
def test_login_success():
    """AC-001: 用户输入正确的手机号和密码后登录成功"""
    ...

def test_login_wrong_password():
    """AC-002: 用户输入错误的密码时显示错误提示"""
    ...
```
