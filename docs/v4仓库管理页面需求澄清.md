# 仓库管理页面需求澄清

## 1. 页面定位

仓库管理页面是青投供应链系统中用于管理仓库布局、库位状态和集装箱信息的核心功能页面。

## 2. 核心功能需求

### 2.1 库位可视化展示

**需求描述**：以网格形式展示仓库的区域划分和库位状态，支持多区域、多行多列布局。

**功能要求**：
- 直观展示仓库布局结构
- 实时显示库位状态（重箱/空箱/空位）
- 支持多区域展示
- 响应式布局，适配不同屏幕尺寸

**状态定义**：
| 状态 | 说明 | 视觉表示 |
|------|------|----------|
| 重箱 (heavy) | 装有货物的集装箱 | 蓝色背景 |
| 空箱 (empty) | 空的集装箱 | 绿色背景 |
| 空位 (available) | 可用库位 | 灰色虚线边框 |

**布局规格**：
- 每个区域固定 3×4 = 12 个库位
- 使用网格布局（CSS Grid）
- 最小库位尺寸：60px × 50px

### 2.2 库位操作功能

**需求描述**：支持库位的选择、导入、移动、出库等操作。

#### 2.2.1 选择库位

**功能要求**：
- 单击选择单个库位
- Ctrl/Cmd + 单击实现多选
- 选中库位显示橙色边框高亮
- 右侧面板显示选中库位的详细信息

**交互规则**：
- 点击有箱库位：选中该库位
- 点击空位：触发导入对话框
- 多选时：支持批量操作（出库等）

#### 2.2.2 导入集装箱

**功能要求**：
- 支持文件导入（Excel、TXT）
- 支持手动输入
- 数据验证（箱号格式、重复检查、容量检查）
- 预览和确认机制

**文件格式要求**：

Excel 格式：
| 箱号 | 状态 | 货主 |
|------|------|------|
| ABCD1234567 | 重箱 | 张三 |
| EFGH7654321 | 空箱 | 李四 |

TXT 格式：
```
ABCD1234567 重箱 张三
EFGH7654321 空箱 李四
```

**验证规则**：
- 箱号格式：4位大写字母 + 7位数字（如 ABCD1234567）
- 重复箱号检查：不允许重复导入
- 容量检查：导入数量不能超过可用库位数

**库位分配策略**：
- 优先分配到指定优先区域
- 优先区域满后分配到其他区域
- 自动分配最近可用库位

#### 2.2.3 移动集装箱

**功能要求**：
- 进入移动模式
- 选择源库位（有箱）
- 选择目标空位
- 执行移动操作

**交互流程**：
1. 点击"移动模式"按钮
2. 点击源库位（显示蓝色闪烁边框）
3. 点击目标空位（显示绿色虚线边框）
4. 系统执行移动，更新库位状态
5. 保持移动模式，可继续移动

**业务规则**：
- 源库位必须有箱（重箱或空箱）
- 目标库位必须为空位
- 移动后保持原有状态（重箱/空箱）
- 移动后保持货物信息

#### 2.2.4 出库操作

**功能要求**：
- 支持单个出库
- 支持批量出库（多选后）
- 确认对话框
- 更新库位状态为空位

**交互流程**：
1. 选中一个或多个有箱库位
2. 点击"出库"按钮
3. 确认对话框
4. 执行出库，清空库位数据

#### 2.2.5 编辑货物信息

**功能要求**：
- 修改货主名称
- 修改货物描述
- 修改备注信息

**适用场景**：
- 选中单个有箱库位时可用
- 点击详情面板中的"编辑"按钮

### 2.3 数据筛选与搜索

**需求描述**：支持按状态筛选和关键词搜索库位。

#### 2.3.1 状态筛选

**筛选维度**：
- 全部：显示所有库位
- 重箱：只显示 status = 'heavy' 的库位
- 空箱：只显示 status = 'empty' 的库位
- 空位：只显示 status = 'available' 的库位

**筛选规则**：
- 实时响应，立即生效
- 保持区域卡片可见，只隐藏不匹配的库位
- 不改变布局结构

#### 2.3.2 关键词搜索

**搜索范围**：
- 集装箱号（containerId）
- 货主名称（cargoInfo.owner 或 container.cargo.owner）

**搜索规则**：
- 不区分大小写
- 模糊匹配（包含关键词即可）
- 跨区域搜索（不受区域限制）
- 300ms 防抖延迟

**搜索结果展示**：
- 匹配的库位显示橙色高亮边框
- 区域标题显示匹配数量（如 "A区 (3)"）

#### 2.3.3 组合筛选

**规则**：
- 状态筛选和关键词搜索可同时使用
- 逻辑关系：AND（交集）
- 例如：筛选"重箱" + 搜索"张三" = 显示货主为张三的重箱库位

### 2.4 统计分析功能

**需求描述**：展示仓库的整体使用情况和统计数据。

#### 2.4.1 全局统计

**统计指标**：
| 指标 | 说明 | 计算方式 |
|------|------|----------|
| 库位总数 | 仓库总库位数量 | 所有区域库位数之和 |
| 利用率 | 仓库使用百分比 | (已用库位 / 总库位) × 100% |
| 已用库位 | 有箱的库位数量 | 重箱 + 空箱 |
| 剩余库位 | 空位数量 | status = 'available' 的库位 |
| 重箱数量 | 重箱库位数量 | status = 'heavy' 的库位 |
| 空箱数量 | 空箱库位数量 | status = 'empty' 的库位 |

**可视化展示**：
- 利用率进度条（颜色随数值变化）
  - 利用率 ≤ 90%：蓝色（正常）
  - 利用率 > 90%：红色（警告）
- 标签式状态分布

#### 2.4.2 区域统计

**统计指标**：
- 区域利用率
- 趋势指示器（上升/下降箭头）
- 匹配数量（搜索时显示）

**可视化展示**：
- 圆形仪表盘（直径 40px）
- 颜色规则同全局统计

#### 2.4.3 实时更新

**更新时机**：
- 库位状态变化（入库、出库、移动）
- 批量导入完成
- 数据刷新

**更新方式**：
- 自动重新计算统计数据
- 触发 Vue 响应式更新
- UI 自动刷新

## 3. 数据模型需求

### 3.1 核心实体

#### 3.1.1 仓库（Warehouse）

**必需字段**：
- id: 仓库ID
- name: 仓库名称
- layout: 仓库布局
- statistics: 统计信息

#### 3.1.2 库位（Slot）

**必需字段**：
- id: 库位ID
- zoneId: 所属区域ID
- coordinates: 坐标位置（row, col）
- status: 状态（heavy/empty/available）
- containerId: 集装箱ID（可选）
- cargoInfo: 货物信息（可选）
  - owner: 货主
  - inboundTime: 入库时间

#### 3.1.3 集装箱（Container）

**必需字段**：
- id: 集装箱ID
- status: 状态
- cargo: 货物信息
  - owner: 货主
- timeline: 时间线
  - inboundAt: 入库时间
  - daysInStorage: 在库天数

### 3.2 数据结构

#### 3.2.1 仓库布局

**结构**：
```
Warehouse
  └─ layout
      └─ rows: SubZone[][]  // 二维数组：行 → 区域
          └─ SubZone
              ├─ id: 区域ID
              ├─ name: 区域名称
              ├─ utilization: 利用率
              └─ slots: Slot[]  // 库位数组（12个）
```

**布局规格**：
- 每个仓库包含多个区域
- 每个区域包含 12 个库位（3×4 网格）
- 支持多行多列布局

## 4. 交互需求

### 4.1 页面布局

**布局结构**：
- **左侧面板**：仓库布局展示区域
- **右侧面板**：详情和统计面板

**响应式要求**：
- 适配桌面端（≥ 1200px）
- 适配平板端（768px - 1199px）
- 适配移动端（< 768px）

### 4.2 视觉反馈

#### 4.2.1 状态颜色

| 状态 | 背景色 | 边框 | 文字颜色 |
|------|--------|------|----------|
| 重箱 | 蓝色（#409eff） | 实线蓝色 | 白色 |
| 空箱 | 绿色（#67c23a） | 实线绿色 | 白色 |
| 空位 | 浅灰（#f5f7fa） | 虚线灰色 | 灰色 |

#### 4.2.2 交互状态

| 状态 | 触发条件 | 视觉效果 |
|------|----------|----------|
| 选中 | 单击库位 | 橙色边框 + 阴影 |
| 搜索匹配 | 搜索关键词匹配 | 橙色高亮边框 |
| 移动源 | 移动模式下的源库位 | 蓝色闪烁边框 + 脉冲动画 |
| 移动目标 | 移动模式下的目标空位 | 绿色虚线边框 + 浅绿背景 |

### 4.3 操作反馈

**成功操作**：
- 显示成功提示消息
- 更新库位状态
- 更新统计信息

**失败操作**：
- 显示错误提示消息
- 回滚操作（如需要）
- 保持原状态

**确认操作**：
- 出库、删除等操作需确认对话框
- 显示操作详情
- 提供取消选项

## 5. 性能需求

### 5.1 响应时间

| 操作 | 响应时间要求 |
|------|--------------|
| 页面加载 | < 2s |
| 数据更新 | < 500ms |
| 操作响应 | < 200ms |
| 搜索响应 | < 300ms（防抖后） |

### 5.2 处理能力

| 场景 | 能力要求 |
|------|----------|
| 库位数量 | 支持 > 1000 个库位 |
| 批量导入 | 支持 > 100 条记录 |
| 实时搜索 | 响应时间 < 300ms |

### 5.3 优化策略

**渲染优化**：
- 使用 computed 属性缓存计算结果
- 防抖搜索（300ms）
- 批量数据处理

**网络优化**：
- 批量 API 请求
- 本地状态优先
- 增量更新

**内存优化**：
- 及时清理定时器和事件监听器
- 避免内存泄漏

## 6. 业务规则

### 6.1 库位分配规则

**优先级顺序**：
1. 优先区域（配置指定）
2. 其他区域（按顺序）

**分配策略**：
- 自动分配最近可用库位
- 支持手动指定库位
- 容量不足时提示警告

### 6.2 状态转换规则

**允许的转换**：
- 空位 → 重箱（入库）
- 空位 → 空箱（入库）
- 重箱 → 空位（出库）
- 空箱 → 空位（出库）
- 重箱 → 空位 → 重箱（移动）
- 空箱 → 空位 → 空箱（移动）

**禁止的转换**：
- 重箱 ↔ 空箱（不允许直接转换）

### 6.3 数据验证规则

**箱号验证**：
- 格式：4位大写字母 + 7位数字
- 示例：ABCD1234567
- 不允许重复

**容量验证**：
- 导入数量 ≤ 可用库位数
- 超出时显示错误提示

**状态验证**：
- 只允许 heavy、empty、available 三种状态
- 状态值不区分大小写（重箱/HEAVY/heavy 都有效）

- Chrome ≥ 90
- Firefox ≥ 88
- Safari ≥ 14
- Edge ≥ 90

- 需要登录认证
- 不同角色有不同操作权限

**数据安全**：
- 数据传输加密（HTTPS）
- 敏感信息脱敏显示
