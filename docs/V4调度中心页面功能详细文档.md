# 调度中心页面功能详细文档

## 一、页面整体布局

页面采用 **12栏栅格布局**，左侧8栏为主操作区，右侧4栏为智能调度与详情区。

```
┌──────────────────────────────────────┬────────────────────┐
│  左侧主操作区 (lg:col-span-8)        │  右侧面板 (lg:col-span-4) │
│  ┌────────────────────────────────┐  │  ┌──────────────────┐  │
│  │  TaskFilterPanel (筛选操作栏)   │  │  │ SmartDispatchEngine│  │
│  └────────────────────────────────┘  │  └──────────────────┘  │
│  ┌────────────────────────────────┐  │  ┌──────────────────┐  │
│  │  TaskList (任务卡片列表)        │  │  │ 任务详情卡片      │  │
│  │  └─ TaskPoolCard × N          │  │  └──────────────────┘  │
│  │  └─ 分页器                     │  │  ┌──────────────────┐  │
│  └────────────────────────────────┘  │  │ 最近识别箱号      │  │
│                                      │  └──────────────────┘  │
│                                      │  ┌──────────────────┐  │
│                                      │  │ DriverRecommendation│ │
│                                      │  └──────────────────┘  │
└──────────────────────────────────────┴────────────────────┘
```

右侧面板在大屏幕下使用 `sticky` 定位，跟随滚动。

## 二、功能模块详解

### 1. 任务筛选与操作栏（TaskFilterPanel）

**源文件**: `apps/frontend/src/components/dispatch/TaskFilterPanel.vue`

| 功能 | 说明 |
|------|------|
| 关键词搜索 | 支持按箱号、客户名、司机名、车牌号、任务号搜索，300ms 防抖 |
| 日期范围筛选 | 日期区间选择器，支持快捷选项（近7天/近30天等） |

| 新建任务 | 点击打开新建调度任务弹窗 |
| 手动刷新 | 刷新任务列表，更新最后刷新时间 |

### 2. 任务列表（TaskList + TaskPoolCard）

**源文件**: `apps/frontend/src/components/dispatch/TaskList.vue`、`apps/frontend/src/components/TaskPoolCard.vue`

| 功能 | 说明 |
|------|------|
| 任务卡片展示 | 每张卡片显示：任务号、创建时间、加急标签、状态标签、路线（起运地→途径点→目的地）、箱号/箱型/封号、司机/车牌、客户、操作按钮 |
| 途径点展示 | 卡片路线区域支持显示途径点，起运地→途径点→目的地完整链路 |
| 分页 | 支持 10/20/50 条/页，含总条数、页码跳转 |
| 空状态 | 无任务时显示空状态提示和"新建任务"按钮 |
| 任务选中 | 点击卡片选中任务，右侧显示详情，选中态蓝色边框高亮 |
| 复选框选择 | 支持通过复选框多选任务，用于批量操作 |
| 任务操作 | 编辑、删除、派单/改派快捷操作 |
| 锁定状态 | 任务被其他用户编辑时显示锁定态（半透明+禁用操作），显示锁定人信息 |

**任务状态及颜色映射**:

| 状态 | 含义 | 颜色 |
|------|------|------|
| pending | 待分配 | 橙色 (warning) |
| transit | 运输中 | 蓝色 (primary) |
| completed | 已完成 | 绿色 (success) |
| exception | 异常 | 橙色 (warning) |
| overtime | 超时 | 红色 (danger) |
| assigned | 已派单 | 蓝色 |

> **改派说明**：当任务状态为 `assigned`（已派单）时，卡片操作按钮显示"改派"而非"派单"，允许重新分配司机。

### 7. 智能推荐（DriverRecommendation）

**源文件**: `apps/frontend/src/components/dispatch/DriverRecommendation.vue`

| 功能 | 说明 |
|------|------|
| 智能推荐按钮 | 选中任务后可点击，执行推荐算法 |
| 推荐结果展示 | 最多展示前5个推荐方案，含司机头像、姓名、车牌、状态、距离、综合评分 |
| 首选标识 | 第1名标注"推荐首选"，蓝色边框高亮 |
| 推荐理由 | 第1名展开显示评分明细：距离得分、负载得分、空闲得分 |
| 一键派单 | 点击推荐结果直接派单，二次确认后执行 |
| 查看更多 | 跳转司机列表弹窗 |

**推荐算法核心**（`apps/frontend/src/services/dispatch/dispatchStrategyService.ts`）：

- **距离优先策略**：距离权重最高
- **成本优先策略**：负载权重最高
- **综合优化策略**：三维度均衡权重
- 评分公式：`总分 = 距离分 + 负载分 + 空闲分`
- 距离计算：Haversine 公式（地球曲面距离）
- 推荐缓存：`apps/frontend/src/services/dispatch/dispatchCacheService.ts`，TTL 过期机制 + 最大容量限制

## 四、服务层架构

| 服务文件 | 职责 |
|----------|------|
| `services/dispatchService.ts` | 核心调度服务：任务CRUD、派单、草稿保存、锁定/解锁、箱号回填、常用地址管理、车辆/司机查询、格式校验 |
| `services/smartDispatchService.ts` | 智能派单算法：Haversine距离计算、车辆评分（距离60%+负载30%+空闲10%） |
| `services/dispatch/dispatchStrategyService.ts` | 多策略调度：距离优先/成本优先/综合优化，可配置权重 |
| `services/dispatch/dispatchCacheService.ts` | 推荐结果缓存：TTL过期、最大容量限制、LRU清理 |
| `services/dispatch/taskAssignmentService.ts` | 任务分配服务：分配/取消/验证/批量分配 |

> `dispatchService.ts` 同时导出一个聚合对象 `dispatchService`，包含所有上述函数的引用。

| 操作 | 说明 |
|------|------|

| `selectAllTasks()` / `deselectAllTasks()` | 全选/取消全选当前页 |

### 任务管理

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取任务列表 | GET | `/api/orders` | 分页+筛选+搜索 |
| 创建任务 | POST | `/api/orders` | 新建调度任务 |
| 更新任务 | PUT | `/api/orders/:id` | 编辑调度任务 |
| 删除任务 | DELETE | `/api/orders/:id` | 删除单个任务 |
| 批量删除 | POST | `/api/orders/batch-delete` | 批量删除任务 |
| 派单 | POST | `/api/orders/:id/assign` | 分配司机和车辆 |
| 任务计数 | GET | `/api/orders/counts` | 各状态任务数量 |

### 任务协作锁定

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 锁定任务 | POST | `/api/orders/:id/lock` | 编辑前锁定，防止多人同时编辑 |
| 解锁任务 | POST | `/api/orders/:id/unlock` | 编辑完成后解锁 |
| 检查锁状态 | GET | `/api/orders/:id` | 查询任务是否被他人锁定 |

### 箱号回填

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 回填箱号 | PUT | `/api/orders/:id` | 司机端回填箱号和封号 |
| 待回填任务 | GET | `/api/orders` | 查询司机待回填箱号的运输中任务 |

### 校验工具

| 函数 | 说明 |
|------|------|
| `validateContainerNo()` | 箱号格式校验（4位大写字母+7位数字） |
| `validateSealNo()` | 封号格式校验 |
| `validatePlateNumber()` | 车牌号格式校验（支持带/不带分隔符） |
| `validatePhone()` | 手机号格式校验（1开头11位） |
| `formatPlateNumber()` | 车牌号格式化（自动添加分隔符 `·`） |

数据适配层使用 `adaptOrderToBackend` / `adaptOrderFromBackend` 进行前后端字段格式转换（camelCase ↔ snake_case）。
