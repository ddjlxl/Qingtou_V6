## 五、Nginx 配置详情

### 5.1 站点配置（/etc/nginx/sites-enabled/）

```nginx
server {
    listen 80;
    server_name _;

    root /var/www/qingtou;
    index index.html;

    gzip on;
    gzip_types text/plain text/css application/json application/javascript
               text/xml application/xml application/xml+rss text/javascript
               image/svg+xml;
    gzip_min_length 256;

    # 前端 SPA 路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
    }

    # OCR 代理
    location /ocr/ {
        proxy_pass http://127.0.0.1:5000/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

## 七、运维操作

### 7.1 服务管理

```bash
# API 服务
sudo systemctl status qingtou-api      # 查看状态
sudo systemctl restart qingtou-api     # 重启
sudo systemctl stop qingtou-api        # 停止
sudo systemctl start qingtou-api       # 启动
sudo journalctl -u qingtou-api -f      # 查看实时日志

# OCR 服务
sudo systemctl status paddleocr        # 查看状态
sudo systemctl restart paddleocr       # 重启
sudo journalctl -u paddleocr -f        # 查看实时日志

# Nginx
sudo nginx -t                          # 测试配置
sudo systemctl reload nginx            # 重载配置
sudo systemctl restart nginx           # 重启
```

```bash

git pull                              # 拉取最新代码
sudo systemctl restart qingtou-api    # 重启服务
sudo journalctl -u qingtou-api -f     # 检查日志

# 前端更新
# 1. 本地构建：pnpm build
# 2. 将 dist/ 目录上传到服务器
# 3. 解压到 /var/www/qingtou/
```

## 十、安全注意事项

| 项目 | 状态 | 建议 |
|------|------|------|
| **UFW 防火墙** | ❌ 未启用 | 建议开启 UFW，仅开放 22、80、443 端口 |
| **SSH 密码登录** | ⚠️ 使用密码 | 建议配置 SSH 密钥登录，禁用密码登录 |
| **腾讯云安全组** | - | 请检查云控制台安全组规则，限制 IP 访问范围 |
| **Nginx 仅 HTTP** | ⚠️ 未配置 HTTPS | 建议配置 Let's Encrypt 免费 SSL 证书 |
| **API 未限 IP** | ⚠️ 公网可访问 | Nginx 反向代理已对内，但 /api/docs 对外暴露 |

### 推荐的安全加固操作

```bash
# 1. 配置 SSH 密钥登录
ssh-keygen -t ed25519
ssh-copy-id ubuntu@43.163.118.22

# 2. 启用 UFW
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable

# 3. 禁用密码登录（配置密钥后）
# 编辑 /etc/ssh/sshd_config，设置 PasswordAuthentication no
# sudo systemctl restart sshd

# 4. 配置 HTTPS（使用 certbot）
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```
