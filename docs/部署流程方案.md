## 二、推荐部署方案

### 方案对比

| 方案 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **方案 A：手动部署（当前方式）** | 继续沿用现有模式 | 简单直接，已跑通 | 手动操作多，容易出错 |
| **方案 B：半自动化脚本部署** | 1-2 人小团队 | 减少重复劳动，可控 | 需要写脚本维护 |

**推荐方案 B** → 写几个部署脚本，在现有手动流程基础上加一层自动化，成本和收益最平衡。

### 3.2 后端部署流程

#### 步骤 1：连接服务器

```bash
ssh ubuntu@43.163.118.22
```

#### 步骤 2：更新代码

```bash
cd /var/www/qingtou-api

# 方式一：直接通过 Git 拉取（推荐，项目根目录已经有 .git）
git pull origin main

# 方式二：如果 git 未配置远程仓库，先用 scp 上传
# 在本地打包：
#   cd apps/server && tar -czf server.tar.gz --exclude=venv --exclude=__pycache__ .
#   scp server.tar.gz ubuntu@43.163.118.22:/var/www/qingtou-api/
```

#### 步骤 3：激活虚拟环境并安装依赖

```bash
cd /var/www/qingtou-api
source venv/bin/activate

# 如果有新增依赖
pip install -r requirements.txt
```

#### 步骤 4：配置环境变量

```bash
cd /var/www/qingtou-api

# 检查 .env 是否存在
ls -la .env

# 如果不存在，从 .env.example 复制并修改
cp .env.example .env
vim .env
```

**.env 关键配置项：**

```ini
# 数据库连接（根据实际情况填写）
DATABASE_URL=postgresql+asyncpg://用户名:密码@数据库地址:5432/qingtou_v6

# JWT 密钥（生产环境务必修改）
JWT_SECRET=你的强随机密钥

# CORS 域名（生产环境必填）
CORS_ORIGINS=["http://43.163.118.22","http://localhost:9527"]
```

#### 步骤 5：执行数据库迁移

```bash
cd /var/www/qingtou-api
source venv/bin/activate
alembic upgrade head
```

#### 步骤 6：重启服务

```bash
sudo systemctl restart qingtou-api
sudo journalctl -u qingtou-api -n 30 -f   # 观察日志确认启动正常
```

### 3.3 前端部署流程

#### 步骤 1：本地构建

```bash
# 在本地开发环境
cd apps/frontend
pnpm install
pnpm build    # 产物在 apps/frontend/dist/
```

#### 步骤 2：上传到服务器

```bash
# 方式一：直接上传（文件较多时较慢）
scp -r apps/frontend/dist/* ubuntu@43.163.118.22:/var/www/qingtou/

# 方式二：打包上传（推荐）
cd apps/frontend
tar -czf dist.tar.gz dist/
scp dist.tar.gz ubuntu@43.163.118.22:~/
ssh ubuntu@43.163.118.22 "sudo tar -xzf ~/dist.tar.gz -C /var/www/qingtou/ && rm ~/dist.tar.gz"
```

> Nginx 指向 `/var/www/qingtou/`，上传后立即生效，无需重启。

### 3.4 一键部署脚本（推荐）

创建部署脚本，减少重复操作。

#### 后端部署脚本

在项目根目录创建 `scripts/deploy-server.sh`：

```bash
#!/bin/bash
# 后端部署脚本
# 用法：bash scripts/deploy-server.sh

set -e

SERVER="ubuntu@43.163.118.22"
REMOTE_DIR="/var/www/qingtou-api"

echo "=== 打包后端代码 ==="
cd apps/server
tar -czf /tmp/qingtou-server.tar.gz \
  --exclude=venv \
  --exclude=__pycache__ \
  --exclude=.pytest_cache \
  --exclude=*.egg-info \
  . > /dev/null 2>&1

echo "=== 上传到服务器 ==="
scp /tmp/qingtou-server.tar.gz $SERVER:/tmp/

ssh $SERVER << 'CMD'
  set -e
  echo "=== 解压代码 ==="
  sudo tar -xzf /tmp/qingtou-server.tar.gz -C $REMOTE_DIR

  echo "=== 安装依赖 ==="
  cd $REMOTE_DIR
  source venv/bin/activate
  pip install -r requirements.txt -q

  echo "=== 执行数据库迁移 ==="
  alembic upgrade head

  echo "=== 重启服务 ==="
  sudo systemctl restart qingtou-api
  sleep 2
  sudo systemctl status qingtou-api --no-pager | head -5

  echo "=== 清理 ==="
  rm /tmp/qingtou-server.tar.gz
CMD

echo "=== 后端部署完成 ==="
```

#### 前端部署脚本

在项目根目录创建 `scripts/deploy-frontend.sh`：

```bash
#!/bin/bash
# 前端部署脚本
# 用法：bash scripts/deploy-frontend.sh

set -e

SERVER="ubuntu@43.163.118.22"
REMOTE_DIR="/var/www/qingtou"

echo "=== 构建前端 ==="
cd apps/frontend
pnpm install
pnpm build

echo "=== 打包构建产物 ==="
tar -czf /tmp/qingtou-dist.tar.gz dist/

echo "=== 上传到服务器 ==="
scp /tmp/qingtou-dist.tar.gz $SERVER:/tmp/

ssh $SERVER << 'CMD'
  set -e
  echo "=== 解压到部署目录 ==="
  sudo tar -xzf /tmp/qingtou-dist.tar.gz -C $REMOTE_DIR
  
  echo "=== 修正权限 ==="
  sudo chown -R www-data:www-data $REMOTE_DIR
  
  echo "=== 清理 ==="
  rm /tmp/qingtou-dist.tar.gz
  
  echo "=== 验证 ==="
  ls -la $REMOTE_DIR
CMD

echo "=== 前端部署完成 ==="
```

### 3.5 数据库首次部署

如果当前服务器还未配置数据库，以下是 PostgreSQL 的部署方案：

#### 方案一：Docker 部署（推荐）

```bash
# 安装 Docker（如果还未安装）
sudo apt install docker.io docker-compose -y

# 启动 PostgreSQL 容器
docker run -d \
  --name qingtou-postgres \
  -e POSTGRES_USER=qingtou \
  -e POSTGRES_PASSWORD=你的密码 \
  -e POSTGRES_DB=qingtou_v6 \
  -p 5432:5432 \
  -v qingtou-pgdata:/var/lib/postgresql/data \
  postgres:16

# 验证
docker ps -a | grep qingtou-postgres
docker logs qingtou-postgres --tail 20
```

#### 方案二：使用腾讯云数据库

如果服务器是腾讯云，可以考虑使用云数据库 PostgreSQL：

1. 在腾讯云控制台购买 PostgreSQL 实例
2. 获取内网连接地址
3. 在安全组中放行 5432 端口
4. 修改 `.env` 中的 `DATABASE_URL`

### 3.6 从零到一的完整部署清单

如果要在**全新服务器**上完整部署 Qingtou V6：

| 步骤 | 操作 | 命令/说明 |
|------|------|-----------|
| 1 | 安装基础环境 | `sudo apt update && sudo apt install -y python3 python3-pip python3-venv nginx git curl` |
| 2 | 安装 Node.js | `curl -fsSL https://deb.nodesource.com/setup_20.x \| sudo -E bash - && sudo apt install -y nodejs` |
| 3 | 安装 pnpm | `sudo corepack enable && corepack prepare pnpm@latest --activate` |
| 4 | 安装 Docker（可选） | `sudo apt install -y docker.io docker-compose` |
| 5 | 拉取代码 | `git clone <仓库地址> /var/www/qingtou-api` |
| 6 | 配置 PostgreSQL | Docker 部署或使用云数据库 |
| 7 | 后端环境配置 | 创建虚拟环境、安装依赖、配置 `.env` |
| 8 | 数据库迁移 | `alembic upgrade head` |
| 9 | 配置 systemd 服务 | 创建 `qingtou-api.service` |
| 10 | 构建前端 | 本地 `pnpm build` 后上传 |
| 11 | 配置 Nginx | 写入站点配置 |
| 12 | 启动服务 | 启动 Nginx + API 服务 |
| 13 | 验证 | 访问页面 + API 健康检查 |
| 14 | 安全加固 | 配置 HTTPS、防火墙、SSH 密钥 |
