## 4. 安全改进建议

### 4.1 马上可做

| 项目 | 操作 | 难度 |
|------|------|------|
| 🔑 **SSH 密钥登录** | 生成密钥对，关闭密码登录 | 低 |
| 🛡️ **UFW 防火墙** | 只开放 22、80、443 端口 | 低 |
| 📦 **PostgreSQL 移除端口映射** | 改为容器内连接（`--network host` 或 Docker 内部 DNS） | 低 |

### 4.2 中期建议

| 项目 | 说明 |
|------|------|
| `.env` 文件权限加固 | `chmod 600 /var/www/qingtou-api/.env`，仅应用用户可读 |
| **数据库连接白名单** | PostgreSQL 的 `pg_hba.conf` 限制只允许应用服务器 IP 连接 |
| **HTTPS** | 申请 Let's Encrypt 证书，全站 HTTPS，避免 API 明文传输密码 |
| **日志脱敏** | 排查是否在 systemd journal / 应用日志中记录了密码 |

### 4.3 长期建议

| 项目 | 说明 |
|------|------|
| **凭据管理** | 使用 1Password / Bitwarden 等密码管理器管理服务器凭据 |
| **敏感信息不入代码** | `.env` 永不提交，临时脚本用完即删 |
| **审计日志** | 开启 PostgreSQL 审计日志，记录所有 DDL 和敏感操作 |
