## 问题描述

本地开发环境登录失败，浏览器报错 `502 Bad Gateway`。服务器环境登录正常。

**期望行为**：本地开发环境应能正常登录。

## 复现步骤

1. 启动前端开发服务器 `pnpm dev`
2. 访问 `http://localhost:9527`
3. 输入用户名密码点击登录
4. 浏览器控制台报错：`Failed to load resource: the server responded with a status of 502 (Bad Gateway)`

## 根因分析

问题呈链式结构，共 4 层：

| 层级 | 问题 | 影响 |
|------|------|------|
| ① | 后端服务未启动 | Vite 代理 `/api` → `localhost:8000` 无响应 → 502 |
| ② | 缺少 `apps/server/.env` | FastAPI 启动后数据库 URL 是默认占位符，无法连接 |
| ③ | 数据库缺 `qingtou` 角色 | 本地 PostgreSQL 只有 `ddjlxl` 和 `postgres`，没有 `qingtou` |
| ④ | 用户密码不一致 | 种子脚本创建的密码是 `admin123`，与 `.credentials.local` 不匹配 |

**根本原因**：本地开发环境缺少完整的初始化配置，`.env` 文件、数据库角色、用户密码三者未同步。

```sql

-- 授权数据库
GRANT ALL PRIVILEGES ON DATABASE qingtou_v6 TO qingtou;

-- 授权 schema 和表

GRANT ALL ON ALL SEQUENCES IN SCHEMA public TO qingtou;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON TABLES TO qingtou;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON SEQUENCES TO qingtou;
```
