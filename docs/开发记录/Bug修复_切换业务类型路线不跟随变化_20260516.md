# Bug 修复报告

## 基本信息
- **Bug ID**：BUG-20260516-001
- **级别**：P1 - 严重
- **报告日期**：2026-05-16
- **修复日期**：2026-05-16
- **关联 AC**：AC-007

## 问题描述
在新建任务页面中切换业务类型后，路线规划（起运地、途径点、目的地）不能跟随变化。用户选择第一个业务类型后，系统自动填充了路线规划，但当切换到另一个业务类型时，路线规划保持不变，没有更新为新业务类型的模板数据。

**期望行为**：切换业务类型时，应该用新业务类型的模板数据覆盖所有路线字段（起运地、途径点、目的地）。

## 复现步骤
1. 打开新建任务弹窗
2. 选择业务类型"重箱运输"，系统自动填充路线：上海港 → 苏州物流园 → 昆山工厂
3. 切换业务类型为"空箱运输"
4. 观察路线规划字段：仍然是"上海港 → 苏州物流园 → 昆山工厂"，没有更新为"宁波港 → 杭州仓库"

## 根因分析
在 `useOrderFormWatchers.ts` 的 `watchBusinessType` 函数中，有条件判断 `if (result.originName && !form.originName)`，只有当字段为空时才会填充模板数据。

**问题代码**：
```typescript
if (result.originName && !form.originName) {  // ← 只在字段为空时填充
  form.originName = result.originName
}
```

**根因**：开发者误解了需求 AC-007，认为"用户手动填写的内容不应该被覆盖"。实际上，AC-007 要求切换业务类型时应该用新业务类型的模板数据替换现有值。

## 修复内容
- **修改文件**：`apps/frontend/src/modules/dispatch/composables/useOrderFormWatchers.ts`
- **修改内容**：移除条件判断，切换业务类型时直接覆盖所有路线字段
- **修复策略**：根据 AC-007，切换业务类型时应该用新业务类型的模板数据替换现有值，不管字段是否已有值

**修复前**：
```typescript
if (result.originName && !form.originName) {
  form.originName = result.originName
}
const hasWaypoints = form.waypoints.filter((w) => w.trim()).length > 0
if (!hasWaypoints) {
  form.waypoints = result.waypoints ? [...result.waypoints] : []
}
if (result.destName && !form.destName) {
  form.destName = result.destName
}
```

**修复后**：
```typescript
if (result.originName) {
  form.originName = result.originName
}
form.waypoints = result.waypoints ? [...result.waypoints] : []
if (result.destName) {
  form.destName = result.destName
}
```
