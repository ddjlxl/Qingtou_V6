# Bug 修复报告

## 基本信息
- **Bug ID**：BUG-20260526-001
- **级别**：P2
- **报告日期**：2026-05-26
- **修复日期**：2026-05-26
- **关联 AC**：无（用户体验问题）

## 问题描述
在仓库管理模块的移动模式下，用户选择一个源库位后点击目标空位完成移动操作，移动成功后，源库位的选中状态（蓝色边框和勾选标记）没有被清空，仍然保持高亮显示，给用户造成困扰。

**期望行为**：移动操作成功完成后，应该清除源库位的选中状态，退出移动模式或重置移动状态。

**实际行为**：移动完成后，源库位仍然保持选中状态，用户需要手动点击该库位取消选中。

## 复现步骤
1. 用户在非移动模式下点击某个有箱库位，该库位被选中（显示蓝色边框和勾选标记）
2. 用户点击"移动"按钮进入移动模式
3. 用户点击源库位（可能是刚才选中的那个），设置移动源（显示闪烁动画）
4. 用户点击目标空位完成移动
5. 移动成功，闪烁动画停止，但**蓝色选中边框和勾选标记仍然存在**

## 根因分析
`move()` 函数在移动成功后只清空了 `moveSourceSlot`（移动源库位）和退出了移动模式，但**没有清空 `selectedSlotIds`**（选中状态集合）。

**代码位置**：`apps/frontend/src/modules/warehouse/stores/useWarehouseStore.ts:100-105`

```typescript
async function move(sourceSlotId: string, targetSlotId: string) {
  await warehouseService.move(sourceSlotId, targetSlotId)
  moveSourceSlot.value = null      // ✅ 清空了移动源
  isMoveMode.value = false          // ✅ 退出了移动模式
  await Promise.all([fetchZones(), fetchStatistics()])
  // ❌ 缺少：clearSelection() 或 selectedSlotIds.value = new Set()
}
```

**为什么出错**：开发时只考虑了移动模式的状态管理，忽略了用户可能在进入移动模式前已经选中了库位，导致移动完成后选中状态残留。

## 修复内容
- **修改文件**：`apps/frontend/src/modules/warehouse/stores/useWarehouseStore.ts`
- **修改内容**：在 `move()` 函数中添加 `clearSelection()` 调用
- **修复策略**：在移动成功后立即清空所有选中状态，确保用户界面状态一致

**修改代码**：
```diff
  async function move(sourceSlotId: string, targetSlotId: string) {
    await warehouseService.move(sourceSlotId, targetSlotId)
    moveSourceSlot.value = null
    isMoveMode.value = false
+   clearSelection()
    await Promise.all([fetchZones(), fetchStatistics()])
  }
```

## 预防措施
1. **状态管理规范**：在涉及多个状态的操作（如移动、出库）完成后，必须检查所有相关状态是否需要重置
2. **测试覆盖**：为涉及多状态交互的操作编写场景测试，确保状态转换的完整性
3. **代码审查检查点**：审查时关注"操作完成后是否清空了所有临时状态"
