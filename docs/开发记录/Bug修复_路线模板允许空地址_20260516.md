# Bug 修复报告

## 基本信息
- **Bug ID**：BUG-20260516-001
- **级别**：P2 - 一般
- **报告日期**：2026-05-16
- **修复日期**：2026-05-16
- **关联 AC**：AC-007

## 问题描述
路线模板管理中，起运地和目的地字段强制要求非空，与 AC-007 不一致。AC-007 明确规定"如无历史记录则留空"，但当前实现不允许保存空的起运地或目的地。

**期望行为**：允许起运地和目的地为空，符合 AC-007 的要求。

## 复现步骤
1. 打开路线模板管理弹窗
2. 清空某个业务类型的起运地或目的地
3. 点击保存
4. 后端返回 422 验证错误（Pydantic 验证失败）

## 根因分析
问题根源在于后端 Schema 和数据库模型的约束过于严格：

1. **Schema 层面**：`RouteTemplateUpdate` 强制要求 `origin_name` 和 `dest_name` 非空
2. **数据库层面**：`business_type_routes` 表的 `origin_name` 和 `dest_name` 列设置为 `NOT NULL`

这导致即使前端传入空字符串，后端也会在 Schema 验证阶段拒绝，无法保存到数据库。

## 修复内容

### 1. 修改 Schema（apps/server/app/schemas/dispatch.py）
- `RouteTemplateUpdate`：允许 `origin_name` 和 `dest_name` 为 `None`
- `RouteTemplateResponse`：允许 `origin_name` 和 `dest_name` 为 `None`
- `RouteTemplateItem`：允许 `origin_name` 和 `dest_name` 为 `None`
- 添加 `convert_empty_strings_to_none` 验证器，将空字符串转换为 `None`

### 2. 修改数据库模型（apps/server/app/models/business_type_route.py）
- `origin_name`：从 `Mapped[str]` 改为 `Mapped[str | None]`，`nullable=True`
- `dest_name`：从 `Mapped[str]` 改为 `Mapped[str | None]`，`nullable=True`

### 3. 创建数据库迁移（apps/server/alembic/versions/a2b3c4d5e6f7_allow_null_route_template_addresses.py）
- 使用数据库迁移修改表结构（创建新表 → 复制数据 → 删除旧表 → 重命名）
- 允许 `origin_name` 和 `dest_name` 为 NULL

### 4. 添加测试（apps/server/tests/test_dispatch_api.py）
- `test_update_route_template_with_empty_origin`：测试允许空的起运地
- `test_update_route_template_with_empty_dest`：测试允许空的目的地
