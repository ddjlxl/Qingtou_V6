# Bug 修复报告

## 基本信息
- **Bug ID**：BUG-20260526-001
- **级别**：P1 - 严重
- **报告日期**：2026-05-26
- **修复日期**：2026-05-26
- **关联 AC**：AC-021（业务类型可选填）、AC-007（切换业务类型时路线规划应跟随变化）

## 问题描述
仓库页面选中集装箱点击出库后，任务自动添加到调度中心，但在调度中心的"编辑任务"界面点击"业务信息"后不能自动填充地址。

**期望行为**：
1. 出库时应支持可选填业务类型，业务类型应正确传递到创建的调度任务
2. 调度中心编辑模式下选择业务类型后，应自动填充该业务类型的模板地址

## 复现步骤
1. 库管在仓库页面选中集装箱，点击出库
2. 后端创建调度任务，但 `business_type` 为 `None`（没有传递）
3. 调度中心出现一条"待分配"任务，没有业务类型
4. 调度员点击"编辑任务"，进入编辑模式
5. 调度员在编辑界面选择业务类型（如"重箱运输"）
6. 地址字段不会自动填充（编辑模式下 watcher 被禁用）

## 根因分析

这是一个**两段式根因**，涉及后端和前端各一个问题：

### 根因 1（后端）：出库创建任务时没有传 `business_type`

[warehouse_service.py](file:///e:/Qingtou_V6/apps/server/app/services/warehouse_service.py#L306-L352) 中 `outbound()` 函数：
- 函数签名没有 `business_type` 参数
- `order_data` 只传了 5 个字段，没有 `business_type`
- API 层也没有从请求中提取 `business_type`
- Schema `OutboundRequest` 没有 `business_type` 字段

**结果**：出库创建的调度任务 `business_type` 为 `None`。

### 根因 2（前端）：编辑模式下 `watchBusinessType` 直接 return

[useOrderFormWatchers.ts](file:///e:/Qingtou_V6/apps/frontend/src/modules/dispatch/composables/useOrderFormWatchers.ts#L38-L39) 中：
```typescript
if (options.mode.value !== 'create') return  // ← 编辑模式直接跳过！
```

当调度员在编辑界面选择/切换业务类型时，因为 `mode === 'edit'`，watcher 直接 return，不会调用 `getRouteTemplate` 去获取地址模板。

## 修复内容

### 后端修复

**修改文件**：
1. [apps/server/app/services/warehouse_service.py](file:///e:/Qingtou_V6/apps/server/app/services/warehouse_service.py#L306-L352)
2. [apps/server/app/schemas/warehouse.py](file:///e:/Qingtou_V6/apps/server/app/schemas/warehouse.py#L83-L84)
3. [apps/server/app/api/v1/warehouse.py](file:///e:/Qingtou_V6/apps/server/app/api/v1/warehouse.py#L87-L98)

**修改内容**：
1. `outbound()` 函数签名添加 `business_type: str | None = None` 参数
2. `order_data` 中添加 `"business_type": business_type`
3. `OutboundRequest` schema 添加 `business_type: str | None = Field(None, pattern=r"^(heavy_transport|empty_transport|short_haul)$")`
4. API 层从请求中提取 `business_type` 并传递给 `outbound()`

**修复策略**：按照设计文档要求，业务类型可选填，支持从出库弹窗传递到调度任务。

### 前端修复

**修改文件**：
1. [apps/frontend/src/modules/dispatch/composables/useOrderFormWatchers.ts](file:///e:/Qingtou_V6/apps/frontend/src/modules/dispatch/composables/useOrderFormWatchers.ts#L36-L67)

**修改内容**：
移除 `watchBusinessType` 中的编辑模式限制：
```typescript
// 删除这行：
if (options.mode.value !== 'create') return
```

**修复策略**：编辑模式下调度员选择业务类型时，应该自动填充该业务类型的模板地址，与创建模式行为一致。
