## 问题描述
用户在"运输流水"Tab 点击"下载模板"按钮时，浏览器控制台报错：`Failed to execute 'createObjectURL' on 'URL': Overload resolution failed.`，导致模板文件无法下载。

**期望行为**：点击"下载模板"按钮后，应成功下载 `transport_record_template.txt` 文件。

## 复现步骤
1. 登录系统，进入"车队管理"页面
2. 切换到"运输流水"Tab
3. 点击"下载模板"按钮
4. 浏览器控制台报错，文件未下载

## 根因分析
**问题根源**：`apps/frontend/src/shared/api/client.ts` 中的 `handleResponseSuccess` 函数对所有 HTTP 响应数据进行 `camelcaseKeys` 转换，以统一前后端字段命名风格。

**为什么出错**：
1. 后端 API `/v1/fleet/transport-records/template` 返回的是 `text/plain` 类型的 Blob 响应
2. `handleResponseSuccess` 函数将 Blob 对象当作普通对象处理，调用 `camelcaseKeys` 进行转换
3. `camelcaseKeys` 会遍历 Blob 对象的所有属性并转换键名，破坏了 Blob 对象的内部结构
4. 最终返回的对象不再是真正的 Blob 实例，导致 `window.URL.createObjectURL()` 失败

**影响范围**：所有使用 `responseType: 'blob'` 的文件下载功能都会受此影响。

## 修复内容
- **修改文件**：`apps/frontend/src/shared/api/client.ts`
- **修改内容**：在 `handleResponseSuccess` 函数开头添加 Blob 响应类型检查
- **修复策略**：检查 `response.config?.responseType === 'blob'`，如果是 Blob 响应，直接返回原始数据，不进行 `camelcaseKeys` 转换

**代码变更**：
```typescript
export function handleResponseSuccess<T = unknown>(response: AxiosResponse<ApiResponse<T>>): T {
  // 新增：Blob 响应直接返回，不进行转换
  if (response.config?.responseType === 'blob') {
    return response.data as T
  }

  const body = response.data as unknown as Record<string, unknown>
  // ... 原有逻辑
}
```
